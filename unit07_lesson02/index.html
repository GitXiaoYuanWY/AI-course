🖥️ 一、认识N8N的界面

N8N的核心是"可视化工作流构建区"，它通过节点（Node）连接形成完整流程。在正式操作之前，我们要先熟悉界面布局。

（一）初始界面

1. 工作流

这里保存和管理你创建的所有流程。每个工作流就像一个"自动化项目文件夹"，可以随时打开、修改、执行或暂停。

你可以通过重命名与添加标签的方法来增加工作流的辨识度。

1 / 1

2. 凭证

当N8N需要连接外部平台（如邮箱、AI模型、飞书等）时，需要凭证授权。凭证相当于一把"钥匙"，让N8N安全访问你的应用。从而可以获取信息、保存信息等等。

例如我们的工作流要浏览我们邮箱的时候，我们就需要提供授权码，这就是凭证。

1 / 1

3. 状态

用于查看工作流的运行记录与结果。你可以在这里检查：
• 是否执行成功？
• 执行用了多长时间？
• 哪些节点出错？

1 / 1

（二）内部工作区

点击右上角"create Workflow（创建新工作流）"后，会进入可视化编辑区。

1. 节点区（Node Canvas）

这是核心工作区，你可以拖拽节点并通过连线形成逻辑。

1 / 1

2. 顶部工具栏

包含保存、运行、暂停、调试等按钮。

1 / 1

3. 左侧节点库（Nodes Panel）

提供所有可用的节点分类，例如"触发器""AI""应用""逻辑"。

🔗 二、基础节点介绍

N8N中的每个节点（Node）都代表一个具体动作或事件。节点可以理解为"自动化任务的最小单元"。

理解节点的类比：

如果把我们上节课提到的炒菜看作是一个工作流：

确定今天要吃什么 → 菜市场买菜 → 洗菜 → 备菜 → 炒菜 → 装盘 → 上桌吃饭

其中，洗菜、备菜、炒菜就算是一个单独的节点。

节点的类型：

触发节点：流程的起跑讯号，决定了工作流何时启动，以什么样的方式启动

AI节点：调用外部模型功能，实现文本、图形、视频等内容创作

应用节点：应用节点让N8N连接并操作外部软件系统，比如邮箱、文档、数据库、社交平台等。

（一）触发节点

触发节点决定了工作流"何时启动"。就像是流程的"起跑信号"。

触发节点类型详解

1. 点击触发器

功能：手动点击触发执行工作流

2. 消息触发器

功能：当邮箱收到新邮件或外部程序发送请求时自动触发工作流

举例：当接收到邮件提醒/别人填写表单的时候自动触发工作流

3. 定时触发器

功能：让自动化工作流在特定时间自动运行

举例：每天早上8点自动发送天气提醒邮件

4. HTTP请求触发

功能：外部系统可通过网络请求调用此流程，从而触发工作流

举例：一个网站表单提交后，自动调用N8N更新数据库，从而返回显示出更新了哪些内容

5. 表单被填写时触发

功能：当用户提交表单数据时自动触发流程

举例：学生在报名表中填写信息，系统自动保存到Excel表中

6. 被另一个工作流调用时触发

功能：一个工作流可以调用另一个工作流，在过程中或是结束的时候

举例：一个文生图的自动化流程，写提示词我们用一个工作流，然后用上一个工作流撰写的提示词用来生成图片作为另一个工作流

优点：避免鸡蛋放在一个筐子里，如果有修改的需求，不至于从头到尾全都改一遍

7. 外部应用触发

功能：如飞书、Slack、Telegram等应用发送消息时触发N8N工作流

举例：当有人在群里发送"日报"两个字时，系统自动收集当天工作内容

（二）AI节点

AI节点让N8N能够调用外部AI模型，实现智能化任务。

1. Agent节点

功能：根据提示词与输入，输出结果

  • 选择模型：配置节点所使用哪个AI模型，原生模型可选
  • 记忆：是否开启记忆，让AI记住前文与上下文，默认的记忆对话是5次
  • 外部工具：允许AI在执行过程中使用外部功能，比如网络搜索、生成图片、抓取网页数据等
  • 规范输出格式：是否让节点输出的结果按照我们设定的结构来输出，例如JSON、表格或Markdown结构，方便后续节点读取

2. 模型接入节点

功能：这些节点的功能是"连接外部AI模型或平台"，相当于让n8n能够调用不同的大模型服务

3. 语言处理类节点

功能：不依赖具体品牌的模型，而是用来处理文本的基本任务，比如提取信息、分类、总结

  • Basic LLM Chain：一个基础语言模型链，用于设计提示词（Prompt）和处理简单输入输出
  • Information Extractor：从文本中提取结构化信息（例如从邮件中提取姓名、地址、日期等）
  • Summarization Chain：把长文本自动概括为摘要，提取重点内容
  • Text Classifier：根据规则或模型，把文本分类（例如判断邮件类型、客户意图等）
  • Question and Answer Chain：用于问答型任务，例如根据文档回答问题
  • Sentiment Analysis：分析文本的情绪（正向、负向、中性）
  • Evaluation：用于评估AI模型的表现或结果准确度

（三）应用节点

应用节点让N8N连接并操作外部软件系统，比如邮箱、文档、数据库、社交平台等。它是"沟通桥梁"，让不同系统的数据可以自动流转。

应用类别 示例节点 功能说明
邮箱类 Gmail / QQ邮箱 自动收取邮件、提取附件、自动回复或发送通知
表格类 Google Sheets / Excel 自动写入成绩、统计数据、生成报表
知识管理类 Notion / Evernote 自动创建学习笔记、任务记录、日报卡片
消息通知类 飞书 / Slack / 微信推送 自动发送提醒、群发通知、接收命令消息
文件存储类 Google Drive / OneDrive 自动备份文件、上传日志或报告
社交平台类 Twitter / 小红书 / B站 自动发布内容、更新动态、抓取评论

⚡ 三、一起做一个最简单的工作流

（一）实战案例

使用N8N构建一个每天早上8点定时给自己的邮箱发送一封根据个人品味定制的名称为音乐推荐的邮件。

（二）工作流逻辑分析

  • 每天早上8点:固定的时间自动触发工作流，时区为中国
  • 根据个人品味定制的名称为音乐推荐:需要用到大语言模型根据提示词自动生成
  • 发送到邮箱:将生成的推荐内容通过邮件发送，邮件主题为"音乐推荐"

（三）工作流技术选型

Cron Trigger（每天08:00）
↓
AI Agent（生成音乐推荐列表）
↓
Email Send（发送到邮箱）

以下是AI Agent节点中使用的完整提示词模板 

# Role: 音乐品味分析与推荐专家

## Profile
- author: LangGPT
- version: 1.0
- language: 中文
- description: 你是一位音乐推荐专家，擅长分析用户的音乐口味，并根据其偏好推荐风格相近的音乐专辑。你会结合用户喜欢的歌手，挖掘相似风格、同一时期或精神气质相通的艺术家及其作品，提供带有发行时间、风格流派、以及每首歌曲市场表现的完整推荐报告。

## Skills
1. 熟悉音乐史中70年代至90年代摇滚乐的发展，包括各类子风格如迷幻摇滚、独立摇滚、前卫摇滚等。
2. 能够识别不同歌手的音乐风格特征，分析用户偏好的音乐元素（如旋律、编曲、歌词主题等）。
3. 能根据用户喜好匹配相似风格的专辑，并提供相关背景资料。
4. 提供每张推荐专辑中的主要歌曲市场表现数据（如当年销量、榜单成绩、评论评价）。

## Rules
1. 推荐至少3张专辑，每张专辑应包含：
   - 歌手名称
   - 专辑名称
   - 发行年份
   - 所属风格流派
   - 专辑简介（为何推荐）
   - 各首歌曲名称及其市场反响（如单曲榜成绩、受欢迎程度等）
2. 所有推荐需基于用户喜欢的歌手（Fleetwood Mac、U2、Pink Floyd、Neutral Milk Hotel、Prince）相似或同代风格。

## Workflows
1. 分析用户口味关键词："70年代~90年代"、"摇滚"、"Fleetwood Mac"、"U2"、"Pink Floyd"、"Neutral Milk Hotel"、"Prince"。
2. 识别这些艺术家的核心风格（如：迷幻摇滚、独立民谣、前卫流行、流行摇滚等）。
3. 搜索与之风格相近、同一时期或受其影响的专辑与歌手。
4. 输出包含完整信息的推荐列表，包括每首歌的市场评价。

## Init
请根据以下用户喜好推荐音乐专辑，并详细说明每张专辑的背景、风格、与用户口味的匹配度，以及其中歌曲的市场表现：
- 喜欢的歌手：Fleetwood Mac、U2、Pink Floyd、Neutral Milk Hotel、Prince
- 偏好风格：70年代~90年代的摇滚
