📐 一、作品集网页结构解析

（一）网页模块组成

一个完整的作品集网页由多个功能模块组成，每个模块承担不同的信息展示任务

网页模块结构示意图

模块详解

1. 顶部导航栏

姓名显示 / 菜单按钮 / 跳转链接

2. 个人介绍模块

头像展示 / 姓名与定位 / 个人标签

3. 学习背景

学校信息 / 专业名称 / 学习时间段

4. 专业能力矩阵

能力卡片组合 / 技能描述

5. 项目案例展示

项目图片 / 项目描述 / 时间与标签

6. 联系信息区

邮箱 / 电话 / 位置信息

（二）模块间关联性

导航锚点连接

顶部菜单可快速跳转到对应内容区域，实现页面内的便捷导航

例：点击"项目案例"直接滚动到项目展示部分

视觉层次结构

重要信息在显眼位置，次要信息在页面底部，形成清晰的视觉优先级

姓名在顶部 → 联系方式在底部

内容逻辑流程

从个人介绍到项目案例，遵循陌生人了解你的自然流程

个人介绍 → 学习背景 → 能力展示 → 项目案例

📊 二、项目资料结构化整理

（一）个人信息准备

基础信息

  • 姓名：你的真实姓名
  • 头像：清晰的个人照片（正方形）
  • 自我介绍：30-50字，简明扼要

个人标签

  • 至少3个专业标签
  • 例如："前端开发"、"UI设计"、"数据分析"
  • 标签要简洁准确

（二）教育背景整理

学校信息示例

学校名称： 北京大学
专业名称： 计算机科学与技术
学习时间段： 2020年9月 - 2024年6月

（三）专业能力矩阵

1

能力点选择

  • 选择6-8个核心能力
  • 每个能力用2-3句话描述
2

能力分类

专业技能

HTML/CSS开发
JavaScript编程

软技能

团队协作
项目管理

工具使用

Figma设计
Git版本控制

（四）项目案例准备

项目案例信息结构

项目准备清单

  • 项目名称：简洁明了，如"电商网站设计"
  • 项目封面图：展示项目成果的图片
  • 项目描述：2-3句话说明项目内容和你的贡献
  • 时间节点：如"2023年3月-2023年5月"
  • 技术标签：如"HTML、CSS、JavaScript、Figma"
  • 项目数量：建议2-4个项目

（五）联系方式整理

必要信息

  • 邮箱地址：常用的电子邮箱

可选信息

  • 电话号码：便于联系
  • 微信号：社交联系方式
  • 所在城市：如"北京市"

🔄 三、模块替换机制与内容整合

（一）AI辅助内容生成

提示词模板

请把以下内容转成我网页模板可替换的文本内容，保持简洁、专业，并分成对应模块：

1. 个人简介（一句定位 + 简短介绍）
2. 学习背景（学校 / 专业 / 时间）
3. 专业能力矩阵（按卡片形式拆成多个2-3句的小描述）
4. 项目案例（每个项目生成标题、简介、亮点、时间、标签）
5. 联系方式（邮箱 / 城市 / 微信等）

以下是我的原始内容：
【在这里粘贴准备好的内容】

内容优化原则

  • 简洁专业：用简短的语言表达专业能力
  • 结构清晰：按模块组织内容，便于替换
  • 符合展示需求：文字不宜过多，重点突出

（二）HTML模板替换操作

HTML模板替换示意

替换操作详解

1. 个人介绍替换

<h1>你好，我是 【学生姓名】</h1>
<p>【AI 生成的自我介绍】</p>
<img src="assets/img/学生头像文件名.png" />

2. 学习背景替换

<h2>学习背景</h2>
<p>【学校名称】 · 【专业名称】 · 【学习时间段】</p>

3. 能力矩阵替换

<div class="skill-card">
  <h3>【能力名称】</h3>
  <p>【能力描述】</p>
</div>

4. 项目案例替换

<div class="project-card">
  <img src="assets/img/【项目图片文件名】" />
  <h3>【项目名称】</h3>
  <p>【项目简述】</p>
  <p class="date">【项目时间】</p>
</div>

5. 联系信息替换

<p>Email: 【邮箱地址】</p>
<p>微信：【微信号】</p>
<p>位置：【所在城市】</p>

替换操作流程图

🔧 五、常见问题与解决方案

（一）内容相关问题

❌

问题1：个人介绍过长或过短

现象：自我介绍超过50字或不足30字

原因：内容过于详细或过于简略

✅

解决方案：控制在30-50字，突出核心优势，删除无关信息

❌

问题2：能力描述不一致

现象：有些能力描述很长，有些很短

原因：没有统一描述标准

✅

解决方案：统一描述长度和风格，每个能力用2-3句话描述

❌

问题3：项目信息不完整

现象：项目缺少名称、描述、时间或标签

原因：准备不充分

✅

解决方案：确保每个项目包含名称、描述、时间、标签四要素

（二）技术问题

⚠️

问题1：图片路径错误

现象：网页中图片显示不出来

原因：图片路径不正确

💡

解决方案：检查相对路径，确保 assets/img/ 前缀正确，文件名无误

❌ 错误：<img src="头像.png" />

✅ 正确：<img src="assets/img/avatar.png" />

⚠️

问题2：HTML结构破坏

现象：网页布局错乱

原因：替换时删除了重要的HTML标签

💡

解决方案：替换时保持原有标签结构完整，只替换标签内的内容

❌ 错误：删除了 <div> 或 </div>

✅ 正确：只修改 <div>内容</div> 之间的内容

⚠️

问题3：样式显示异常

现象：网页样式与预期不符

原因：CSS类名被修改或标签未正确闭合

💡

解决方案：检查CSS类名是否保留，标签是否正确闭合

❌ 错误：class="skill-card" 改成了其他名称

✅ 正确：保持 class 属性不变
