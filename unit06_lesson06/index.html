6.AI 网页生成控制实战

掌握结构化提示词 | 实现可控的个性化网页生成

课程概述

课程回顾

  1. 识别并拆解 AI 生成网页的三层结构：HTML（结构）/ CSS（样式）/ JavaScript（行为）
  2. 通过示例阅读 AI 代码，能判断哪些片段属于结构、视觉或交互
  3. 理解同一提示词可能产生不同结果，具备基础的"阅读 AI / 检查 AI"的意识

学习目标

  1. 掌握结构化提示词写法，能对 HTML/CSS/JS 输出进行明确约束与控制
  2. 理解网页"个性化表达"的三方向：视觉风格（CSS）/ 结构风格（HTML）/ 语气风格（文案）
  3. 使用提示词模板生成风格一致、结构清晰、交互合理的页面骨架
  4. 能围绕"极简 / 科技 / 卡通 / 时尚"等风格，设计可复用的风格化提示词模板
  5. 完成一次从"AI 自由生成"到"按规范生成"的提示词迭代实践

课程目录

  1. 网页生成中的提示词控制基础
  2. 页面结构控制：让 AI 严格按所需格式输出

课程章节

深入学习 AI 网页生成的精准控制方法

一、提示词控制基础与结构化表达

（一）AI输出"风格不一致"的原因分析

  • 理解不足：AI是根据已有文字去"找相似说法"，并不真正理解网页规范；结果可能看着像对的，但细节容易不统一
  • 自动补全：提示里少了关键信息时，AI会自己补模块、名字和顺序；这个补法不可控，不同次输出就会不一样
  • 随意发挥：没有规定清楚"要用什么结构和格式"时，AI在结构层级、样式命名和排版细节上会随意发挥，整体看起来就不一致

（二）结构化提示词的五段式骨架

💡 核心思想： 通过明确的五段式结构，给AI提供完整、清晰、无歧义的指令，确保输出的可控性和一致性。

（三）可直接写进提示词的约束要点

指定布局方式

横向排列还是网格排列；移动端如何换行

指定配色变量

把颜色起统一名字（如 --primary），方便全站一致调整

指定命名规则

类名按固定写法（如"模块__元素--状态"）统一起来

指定文件分离

要求分别输出 HTML / CSS / JS，互不混在一起

二、页面结构控制——让 AI 按所需格式生成

（一）用"结构树 + 顺序"说清要什么

（二）写进提示词的关键要求

1

说明约束

不要新增模块、不要改动顺序

2

添加注释

每个 section 开头加一句注释，指明这个区块需要做的是什么

3

规定文字长度

使用示例文字（占位）时的长度规则

例：标题不超过 8 个字；简介 20–40 字

（三）生成后进行查验

顺序检查

是否完全按清单输出，一个不多、一个不少？

标签检查

是否用了上面规定的那些有含义的标签？

定位检查

关键区块是否都有固定 id，能被样式和 JS 精准选中？

控制检查

有没有多出来的模块或内容，属于 AI 自己加的？

三、网页个性化方向一——视觉风格控制（CSS 层）

（一）风格控制的核心抓手

统一管理的优势：

  • 改一次变量，全站同步更新
  • 设计规范清晰，避免随意数值
  • 易于维护和扩展
  • 风格一致性有保障

（二）可复用风格片段（示例）

CSS 变量与基准样式示例 
:root{
  --primary:#2b6cff;
  --text:#222; --muted:#666; --bg:#f7f8fa;
  --radius:12px; --shadow:0 6px 18px rgba(0,0,0,.06);
  --space-1:8px; --space-2:16px; --space-3:24px; --space-4:32px;
}

body{
  color:var(--text);
  background:var(--bg);
}

.section-title{
  font-weight:700;
  margin-bottom:var(--space-2);
}

.card{
  border-radius:var(--radius);
  box-shadow:var(--shadow);
  padding:var(--space-3);
  background:#fff;
}

.btn{
  padding:.6em 1.2em;
  border-radius:8px;
  background:var(--primary);
  color:#fff;
  transition:.25s ease-out;
}

.btn:hover{
  transform:translateY(-2px);
}

提示词锚点（可直接写入提示词）

  • 使用 CSS 变量 统一配色、文字、间距，禁止行内样式
  • 按钮采用轻微悬浮动效，过渡不超过 300ms，使用 ease-out
  • 标题样式统一（字重、字号、下边距），卡片统一圆角与阴影

（三）风格控制检查清单

变量化程度

一处改变量，能否全站换风格？（主色、背景、圆角、阴影是否都走变量）

排版一致性

标题层级清楚，正文字号与行高统一

间距统一性

是否都用 8/16/24/32，避免随意数值

动效克制性

时长 200–300ms，ease-out，无过度动画

四、网页个性化方向二——结构风格控制（HTML 层）

（一）三类结构风格（用途与要点）

扁平化结构

用途：内容导览、信息浏览为主的页面

  • 区块之间留白更大
  • 段落更舒展
  • 强调"扫一眼就懂"

紧凑信息块

用途：产品列表、功能清单、对比信息

  • 卡片密度更高
  • 条目数量固定
  • 强调"同屏展示更多"

叙事式结构

用途：作品集、品牌故事、时间线展示

  • 按时间或故事顺序
  • 每个模块有承接关系
  • 结尾有行动引导

（二）结构风格的提示词约束

定量

明确模块数量与条目数

例："Features 固定 3 条，Testimonials 固定 2 条，不多不少。"

定模版

给每个模块规定字段与字数

例："Work 卡片包含：标题（≤10 字）/ 简介（≤30 字）/ 图片（占位名）。简介只写一行，不换行。"

定边界

禁止新增与目标无关的模块

例："不要生成相关文章、随机推荐、订阅弹窗等与本页无关的内容。"

定标识

关键区块要有稳定 id，顶端加一句注释

例："`section#work` 顶部加入注释：'作品展示区：3 张卡片'。"

定顺序

按给定顺序输出，不得改序

例："严格执行 Header → Hero → About → Work → Testimonials → Contact → Footer 的顺序。"

五、网页个性化方向三——表达风格控制（文案与语气）

（一）语气为何影响"个性化"

文案是页面的"说话方式"，决定给人的第一印象。

语气与视觉相互叠加：简洁的语气配简洁的版式，更统一；温暖的语气配柔和的配色，更贴合。

（二）常见语气类型、用途及特征

正式

企业 / 政务

  • 客观、克制
  • 信息准确
  • 少用感叹
"我们致力于为客户提供高质量的解决方案。"

轻松

个人 / 生活方式

  • 口语化
  • 亲和力强
  • 有生活场景
"嘿，来看看我最近做的小项目！"

科技感

产品 / 创新

  • 短句
  • 精确名词
  • 强调功能与数据
"99.9% 正常运行时间。毫秒级响应。"

情绪化

品牌 / 故事

  • 画面感强
  • 节奏有起伏
  • 比喻适度
"每一个像素，都承载着我们的热爱。"

（三）语气控制要点（写进提示词即可）

标题与副标题

限定句长与结构

如"标题≤10字；副标题为短句或中英对照"

避免项

禁止营销腔、口号化

空洞的形容词堆砌

信息密度

一条只说一个重点

每段只围绕一个主旨

一致性

全站同一种称呼方式

同一标点风格（如统一用"："或"-"）

六、综合示范——可控与个性化的整合生成

（一）综合提示词模板（可直接改写）

类型
单页作品集（Portfolio Landing）
目的
展示 3 个代表作品并引导联系
结构

严格按顺序输出，禁止新增或改序：

  1. header（logo、nav）
  2. section#hero（主标题≤12字、副标题≤24字、主按钮）
  3. section#works（3 张卡片：title≤10字、desc≤30字、img 占位）
  4. section#about（一段 40–60 字说明）
  5. section#contact（表单：name/email/message）
  6. footer（版权/社交链接）
CSS 风格
极简黑白，使用 CSS 变量：--primary/--text/--bg/--radius/--shadow； 统一间距刻度（8/16/24/32），按钮 hover 轻微上浮，过渡 ≤300ms。
命名规范
BEM；禁止行内样式；媒体查询支持 360–1440 宽度。
JS 行为
提交表单时校验必填，按钮点击有轻微动效； 所有 JS 与 CSS 请分段输出，附少量注释。

（二）示范讲解要点

💡 关键要点

  1. 对照输出核查：结构顺序、模块数量、命名规范是否符合要求
  2. 检视风格一致性：变量化程度、间距/字重/阴影是否统一
  3. 行为层合理性：表单校验是否生效、DOM 选择器是否与结构一致
  4. 迭代策略：只改提示词中的变量与约束，不直接改代码，以验证"可控性"

编写个性化网页生成提示词

作业要求

总体目标： 编写一份个性化网页生成提示词，控制 AI 按规范输出结构清晰、风格一致、可复用的页面骨架。

具体要求：

  1. 明确网页类型与目的（如着陆页/商详页/作品集）

  4. 约定命名规范与禁止项（BEM、禁行内样式、禁额外库）
  5. 设定文案与语气（标题句长、信息密度、避免项）
