# **板块二：课程章节**

## 一、Suno 提示词的整体结构框架

AI 音乐生成系统的提示词机制依赖文字输入来构建声音、结构与音乐行为。不同的输入区域承担不同的功能，因此正确理解提示词在系统内部的分工，是后续提示词写作的基础框架。

Suno 的提示词可分为 **三个主要输入区块**：**Description、Lyrics、Title**。这三者协同工作，将抽象的音乐意图转化为具体的声音结果。

三个输入区块在系统内部的生成链路中承担不同职责：

- **Description → 声音与风格骨架**（决定是什么类型的音乐）
- **Lyrics → 结构、段落、是否唱、如何唱**（决定音乐内容如何展开）
- **Title → 信息补充、主题微调**（决定作品的命名与主题氛围）

三者共同构成一条“音乐语言指令链”。此外，Suno 的底层模型是基于英语训练数据为主的，所以它天生更擅长理解英文。因此用英文提示词，AI 才能更精准地理解你要的风格、情绪和音乐结构。

## 二、曲风提示词写法（Description 部分）

Description 区域是 Suno 生成音乐时最重要的风格定义入口。系统依赖该区域的语言描述，构建音乐的整体风格、声场结构、节奏强度与情绪走向。提示词写作的核心任务之一，即是将抽象的创作逻辑（例如适用于毕业短片、适合年轻听众、想表达怀旧感等）转换为具体可执行的英文风格描述。

### 标准写法顺序：固定结构让风格更稳定

完整的 Description 写法推荐遵循如下顺序：

> 流派（Genre） + 情绪（Mood） + 场景（Scene） + 节奏/BPM + 主乐器音色（Timbre） + 主题关键词（Theme）
> 

这一顺序的原因在于：

- 系统优先识别 Genre 和 Mood，因此应放在最前；
- 场景通常用于补充氛围；
- BPM 与乐器信息为结构性描述，应置于中段；
- Theme 用于强化音乐语义，可放结尾。

示例结构：

```jsx
Warm nostalgic pop ballad, 95 BPM, soft piano and strings, gentle groove,Memories of Youth and Campus Life。
// 温暖怀旧的流行抒情曲风，95BPM，中慢速节奏，柔和钢琴与弦乐，律动轻柔，适合青春校园回忆主题
```

这一写法具备以下优势：

（1）结构化语言输入让风格更容易准确呈现；

（2）减少系统误判或偏离意图；

（3）便于大量重复创作时保持写作一致性。

## 三、歌曲结构标签（Structure Tags）——控制 AI 的音乐段落布局

在 AI 音乐提示词体系中，结构标签（Structure Tags）承担“结构指挥官”的角色。Description 控制音乐的“风格外形”，而结构标签控制音乐的“段落顺序与内容组织方式”。对于 Suno 而言，结构标签是最直接、最强力的结构控制工具，它们不会被唱出，却会影响旋律线、节奏密度、伴奏编排以及段落之间的衔接方式。

### （一）结构标签的本质作用：让 AI 按既定顺序展开音乐

结构标签具有三个核心功能：

1. **决定音乐段落的排列顺序**
    
    如：
    
    ```
    [intro]  // 前奏 / 开场段
    [verse]  // 主歌（叙述部分）
    [chorus] // 副歌（高潮段）
    ```
    
    系统会严格按照顺序构建音乐结构。
    
2. **决定每个段落的类别与音乐行为**
    
    例如 chorus 通常旋律更高、力度更强；intro 多为空气氛围或简短动机。
    
3. **指定段落不被唱出（标签本身永不发声）**
    
    标签被视为结构指令，不会转化为声音文本。
    
    结构标签是歌词区（Lyrics）的主要控制工具，与真实歌词共存，并以线性顺序主导音乐架构。
    

### （二）常用结构标签及其音乐功能

以下标签为实际创作中使用频率最高、系统识别最稳定的一类。每个标签都带有特定的音乐行为模式。

| 结构标签 | 英文写法 | 功能简介 | 音乐行为特征 | 常见使用位置 |
| --- | --- | --- | --- | --- |
| 序曲 | `[intro]` | 开场氛围、铺垫风格 | 无主唱或弱人声，氛围铺陈、简短动机 | 歌曲开头 |
| 主歌 | `[verse]` / `[verse 1]` / `[verse 2]` | 叙述内容展开 | 旋律较低、节奏平稳，用于讲述故事 | 副歌前的铺垫 |
| 副歌 | `[chorus]` | 歌曲高潮、记忆点 | 旋律提升、节奏加强、情绪最强 | 每轮循环高潮位置 |
| 桥段 | `[bridge]` | 打破主歌副歌循环、提供转折 | 和声或旋律出现明显变化 | 副歌后、尾声前 |
| 尾声 | `[outro]` | 稳定收尾、情绪落点 | 可淡出、弱化、保留部分主题动机 | 歌曲结尾 |
| 间奏 | `[instrumental]` | 无歌词段落、纯乐器表现 | 无主唱，用乐器推动情绪或过渡 | 段落之间、桥段后 |
| 独奏 | `[solo]` / `[guitar solo]` / `[piano solo]` | 展示乐器旋律线、强化亮点 | 根据乐器不同产生旋律或技巧性段落 | 副歌之后、桥段前后 |
| 拆解段 / 下降段 | `[breakdown]` | 将能量降低、准备进入爆发段 | 节奏简化、和声减少、电子元素弱化 | drop 前、桥段前 |
| 落点（爆发段） | `[drop]` | 强烈冲击段落、舞曲核心结构 | 强低频、强鼓点、主音合成器进入 | build-up 之后 |
| Fill（鼓填充） | `[drum fill]` | 段落衔接、增强过渡感 | 短促鼓点滚奏 | verse→chorus / chorus→桥段 |
| Harmonies（和声） | (和声内容) | 增加厚度、补充主旋律 | 轻声层叠、背景声部 | 副歌、情绪高潮段 |
| 关键句重复 | `[hook]` | 强调最关键的一句、提升记忆点 | 重复短句、人声层次更突出 | 副歌中或独立段落 |
| 构建段 | `[build-up]` | 为 drop 做铺垫、增加张力 | 节奏密度逐渐提升、鼓点加快 | drop 前 |
| 结束符 | `[end]` | 明确终止、避免模型延伸 | 立即结束，无尾音拖长 | 最后一行 |

### **（三）结构标签的使用原则（核心）**

结构标签的效果在于“顺序”、“清晰度”、“指令性”。在多次实验中可总结出以下稳定规律。

**1. 使用中括号，标签不会被唱出**

如：

```
[verse] // 主歌
```

只作为结构提示，不会影响歌词内容本身被唱出。

---

**2. 标签顺序 = 实际音乐结构顺序**

顺序错误可能导致结构混乱，例如：

```
[chorus] // 副歌（歌曲高潮段）
[intro]  // 前奏 / 开场段

```

会让副歌提前出现，但前奏后置，造成风格割裂。

---

**3. 可在标签后添加限定词增强控制力**

限定词用于延长、缩短或调整段落属性。

如：

- *brief instrumental*（短）
- *long instrumental*（长）
- *long guitar solo*（较长吉他独奏）

示例：

```
[instrumental] // 纯音乐段落（无歌词不唱）
brief ambient intro // 简短的氛围感前奏（空灵、铺垫情绪）
```

---

**4. 标签可与歌词混写，互不冲突**

示例：

```
[verse 1] // 主歌第1段（叙述开始）
风经过走廊的方向
落叶牵起久违的回忆
```

标签定义结构，歌词提供可唱内容。

---

**5. 可重复使用标签，用于多段循环**

常见流行结构：

```
[verse 1] // 主歌1
[verse 2] // 主歌2
[chorus] // 副歌
[chorus] // 副歌（重复一次）
```

---

### **（四）典型流行音乐结构示例（可直接使用）**

以下结构非常稳定，适合绝大多数流行、民谣、轻摇滚类创作。

```
[intro]   // 前奏 / 开场段
[verse 1] // 主歌第1段（叙述内容开始）
[verse 2] // 主歌第2段（继续叙述）
[chorus]  // 副歌（第一轮高潮）
[bridge]  // 桥段（情绪或旋律转折）
[chorus]  // 副歌（第二轮高潮 / 重复加强）
[outro]   // 尾声 / 收尾段
[end]     // 结束符（明确终止歌曲）
```

该结构包含所有必要的铺垫、高潮、转折与尾声，适合讲故事、抒情或校园回忆主题。

---

### **（五）不同风格的结构示例（进阶）**

为了适应不同创作场景，可根据流派类型使用不同结构。

**1. EDM / Electronic**

特点：多次 build-up 与 drop。

```
[intro]     // 前奏 / 开场段
[build-up]  // 累积能量段（为 drop 做铺垫）
[drop]      // 爆发段 / 强能量落点
[verse]     // 主歌（叙述内容）
[chorus]    // 副歌（高潮段）
[drop]      // 第二次爆发段 / 能量强化
[outro]     // 尾声 / 收尾段
[end]       // 结束符（明确终止歌曲）
```

---

**2. 说唱 Rap**

特点：大量 verse，副歌可短或无。

```
[intro]    // 前奏 / 开场段
[verse 1] // 主歌第1段（叙述开始）
[verse 2] // 主歌第2段（继续叙述）
[hook]    // 关键句重复段（强化记忆点）
[verse 3] // 主歌第3段（进一步叙述或推进情绪）
[outro]   // 尾声 / 收尾段
[end]     // 结束符（明确终止歌曲）
```

---

**3. 电影配乐 Cinematic**

特点：段落叙事感强，但无歌词。

```
[intro]        // 前奏 / 开场段（建立氛围）
[instrumental] // 纯音乐段落（无歌词，以乐器为主）
[build]        // 构建段（逐步增强情绪与能量）
[climax]       // 高潮段（全曲能量最强、最饱满的部分）
[fade out]     // 渐弱收尾（声音逐渐变小，进入结束）
[end]          // 结束符（明确终止整首曲子）
```

### **（六）结构标签如何与 Description 协同工作**

示例说明：

Description：

*Warm nostalgic pop ballad, 95 BPM, soft piano and strings, gentle groove, for青春回忆。*

Lyrics：

```
[intro] brief instrumental // 前奏：简短的纯音乐开场
[verse 1]                 // 主歌第1段（叙述开始）
回忆缓缓落在旧课桌上

[chorus]                  // 副歌（歌曲高潮）
我们一起的时间慢慢走

[outro]                   // 尾声 / 收尾段
instrumental fade out     // 纯音乐淡出收尾

[end]                     // 结束符（明确终止歌曲）

```

配合效果如下：

- Description 定义声音：柔和、校园、怀旧、钢琴弦乐；
- 结构标签决定铺垫、主歌、高潮、尾声的顺序；
- 歌词提供可唱文本；

## 四、歌词提示词写法（Lyrics 部分）

Lyrics 区域承担双重任务：

**（1）提供可被演唱的歌词内容**

**（2）提供结构标签来控制音乐段落行为**

这使得 Lyrics 成为 Suno 中复杂度最高的提示词区域。写法是否清晰直接影响旋律线、人声表现、音乐结构的稳定性。本章分为四部分，系统说明歌词提示词的构成方式、两类“不可唱”标签、和声标记与发音稳定技巧。

### （一）歌词区的两种作用：结构控制 + 可唱内容

Lyrics 的构成可以分为：

**1. 结构标签（Structure Tags）**

如上一章所述，用于控制段落顺序，例如：

```
[intro]    // 前奏 / 开场段
[verse 1] // 主歌第1段（叙述内容开始）
[chorus]  // 副歌（高潮段）
[outro]   // 尾声 / 收尾段

```

标签不会被唱出，但会影响音乐生成逻辑。

**2. 可唱内容（Lyric Lines）**

系统会将标签下方的文本识别为“歌词”并配合旋律演唱。例如：

```
[verse 1] // 主歌第1段（叙述开始）
风吹过旧操场的方向
回忆沉在温暖的阳光里

```

这类文本会被完整唱出。

两者之间的顺序呈现线性结构，系统会依照实际排列来构建整首歌的音乐内容。

### （二）歌词来源的两种方式

Lyrics 可以通过两种方式生成：

---

1. 直接写歌词（AI照着唱）
    
    你提供完整的歌词内容，AI按照你写的文字演唱。
    
    示例：
    
    ```
    [verse] // 主歌（叙述段落）
    在城市的风里听见回忆慢慢亮起
    在路灯的光下写下未完成的自己
    
    ```
    

---

1. 方式二：写“控制标签”（AI不唱字，只执行动作）
    
    你不提供具体歌词，而是用特殊符号来控制音乐结构、节奏、和声等行为。
    
    常见原标签包括：
    
    - `~~~~`（节奏占位符，不唱）
    - `(和声内容)`（括号内会被当成和声）
    - `{不可唱内容}`（描述段落，用于补充行为）
    
    示例：
    
    ```jsx
    [verse]      // 主歌段落
    ~~~~         // 只给节奏，不唱字
    ~~~~
    (ah—ah)      // 这里加入轻柔和声
    {进入情绪更沉稳的编曲}  // 描述段落氛围，不唱
    
    [chorus]     // 副歌段落
    (ooo—ooo)    // 大和声推进
    ~~~~         // 节奏继续推进
    {增强鼓点与背景铺垫}  // 控制编曲，AI不会唱
    ```
    

## 五、提示词一键生成模板（快速生成版）

提示词模板

```jsx
你是一位享誉全球的音乐制作人与作曲家，你深谙如何将抽象的情感与故事转化为动人的旋律。你的任务是基于给定的主题，创作一份专为Suno等文生音乐模型设计的、结构化、富有艺术指导性的"音乐蓝图"（Prompt）。这份蓝图将指引AI生成一首具有强烈故事感、画面感和场景感的原创音乐。

**创作流程与规则：**

这份音乐蓝图由两部分构成：**`歌曲风格 (Style of Music)`** 和 **`歌词 (Lyrics)`**。

#### **第二部分：`歌词 (Lyrics)` - 谱写歌曲的叙事剧本**

这是歌曲的"剧本"，严格遵循Suno可识别的格式，规定故事的起承转合。

**结构标签 (Structural Tags):**

*   `[Intro]` (前奏)
*   `[Verse]` (主歌): 叙事。
*   `[Chorus]` (副歌): 升华主题，情感爆发。
*   `[Bridge]` (桥段): 转折与过渡。
*   `[Outro]` (尾奏)
*   `[Instrumental]` / `[Guitar Solo]` 等: 纯器乐段落。

**格式规则 (至关重要):**

*   **结构清晰:** 每个结构标签（如 `[Verse]`）和其歌词为一个独立单元。
*   **换行分隔:** **使用换行来分隔不同的结构单元和歌词中的不同句子。** 这是Suno识别结构和节奏的唯一方式。**请勿使用分号或句号作为结构分隔符。**

**创作指导:**
*   **叙事弧光：** 歌词应构建一个完整的情感或故事弧线。
*   **诗意简练：** 用精炼的语言勾勒清晰的画面，让每一句歌词都服务于整体的故事和情绪，避免陈词滥调。

---

**最终输出要求：**

你的最终产出必须是一个**纯净、可直接复制的普通文本**，严格包含 `歌曲风格 (Style of Music)` 和 `歌词 (Lyrics)` 两个部分，并使用Markdown标题清晰分隔。请勿添加任何额外的解释或注释。

**标准输出格式示例：**

**歌曲风格 (Style of Music):**
Epic China-Chic, Cinematic Pop, Orchestral, Solemn, Powerful Female Vocals, Choir, Guzheng, Cinematic Drums

**歌词 (Lyrics):**
[Intro]
一道悠远的古筝旋律
渐渐融入现代合成器的节拍

[Verse 1]
黄河的水浪 卷起了篇章
甲骨的刻痕 写满了洪荒
青铜器之上 纹路在流淌
百家争鸣时 思想闪着光

[Chorus]
啊 千年星河 奔流入我
山川与脉搏 是同一首歌
啊 文明的火 从未曾熄落
在时光尽头 回响着承诺

[Outro]
女声悠长的吟唱
在管弦乐和古筝声中
慢慢消散
```
