📖 板块一：从Markdown到JSON

（一）结构化语言的演进

上节课我们学过 Markdown —— 它是一种让文字更整齐、层次更清楚的写作方式。用Markdown，AI能更容易看懂我们想表达的重点和逻辑。

不过，当信息要被电脑程序保存、传送或自动处理时，只靠Markdown还不够。这时，AI需要一种更标准、更结构化的表达方式，让不同系统都能"听懂"同样的数据。这种语言就是 JSON。

（二）Markdown与JSON的关系

Markdown和JSON其实就像是AI沟通的两种语言：

Markdown

写给"人"看的

讲究排版清楚、层次分明

# 今天的计划
**重点复习**

JSON

写给"机器"看的

讲究结构整齐、方便读取

{"日期": "2025-11-06",
"任务": ["学习", "运动"]}
对比内容 Markdown（写给人看） JSON（写给机器看）
谁在用 人类读者 AI或电脑程序
用来干什么 整理文字、做笔记、写文档 存放数据、传递信息
看起来 像普通文本 像一张"数据表"
优点 好读、好看 结构清楚、AI能理解

Markdown → 人告诉AI"要做什么"
JSON → AI告诉系统"我做了什么"

🔍 板块二：认识JSON——AI最懂的语言

（一）JSON的定义与起源

JSON（JavaScript Object Notation） 是一种让电脑看得懂、记得住的文字格式。它最早是一个叫Douglas Crockford的人发明的。

你可以把它想成：电脑之间聊天用的"简单语言"。

（二）JSON的核心特点

轻巧好懂

普通文字文件，不需要复杂软件

写法简单、结构清晰

有条理且有结构

像一张信息表格

每条信息都有"名字"和"内容"

到处都能用

跨平台、跨语言

电脑、网页、手机都支持

内容清楚明白

每条数据都有名字说明

不用猜，一目了然

用途广泛

网站传数据、App设置

AI模型结果输出

示例：记录学生信息

{
  "姓名": "小明",
  "年龄": 18,
  "成绩": 95
}

这里一共三行，每一行：
• 左列是"名称"（比如 姓名、年龄、成绩）
• 右列是"内容"（比如 小明、18、95）

（三）AI为什么离不开JSON

在人工智能工作时，电脑需要不断接收信息、处理信息、再输出结果。如果每个系统说的"数据语言"都不一样，电脑之间就会听不懂、对不上。

示例：AI生成新闻摘要

{
  "标题": "人工智能正在改变教育",
  "摘要": "AI帮助学生更高效学习，也让教学更有针对性。",
  "关键词": ["人工智能", "教育", "学习"]
}

新闻网站可以：

  • ✓ 自动把"标题"当作新闻题目
  • ✓ 把"摘要"放到正文里
  • ✓ 用"关键词"来做分类或搜索

💻 板块三：JSON语法入门

JSON的结构规则与数据类型

1

对象（Object）

  • 用花括号 {} 表示
  • 内部由若干"键–值对"组成
  • 键与值之间用冒号 : 连接
  • 每组信息之间用逗号 , 隔开
{
  "名字": "小明",       // 用{}包起来的是对象
  "年龄": 20,           // "名字"和"年龄"是"键"
  "爱好": ["音乐", "运动", "游戏"]  // 值可以是各种类型
}
2

键（Key）

  • 键是"信息的名字"，必须用双引号 " " 包起来
  • 例如 "名字"、"年龄"
3

值（Value）

值是"具体的内容"，可以是：

字符串
"小明"

文字内容，用双引号包起来

数字
20

不用加引号

布尔值
true / false

表示"是"或"否"

数组
["音乐","运动"]

一组内容，用[]包起来

对象
{ }

嵌套的对象

空值
null

表示"暂时没有内容"

{
  "名字": "小明",                  // 字符串：文字内容
  "年龄": 20,                     // 数字：不用加引号
  "是否学生": true,               // 布尔值：true表示"是"
  "爱好": ["音乐", "运动", "游戏"], // 数组：一组内容
  "家庭": {                       // 对象：嵌套的信息
    "父亲": "小王",
    "母亲": "小李"
  },
  "电话": null                    // 空值：暂时没有内容
}
4

数组（Array）

  • 用方括号 [ ] 包起来
  • 表示一组内容，比如一串爱好、朋友名单等
"爱好": ["音乐", "运动", "游戏"]

表示"爱好"这一项有三个内容

5

逗号规则

  • 每组内容之间要用逗号 ,
  • 最后一组后面不能加逗号
错误示例
{
  "名字": "小明",
  "年龄": 20,   ← 最后一行不能有逗号
}
正确示例
{
  "名字": "小明",
  "年龄": 20    ← 最后一行不加逗号
}

🤖 板块四：AI结构化输出实践

结构化输出是AI实用化的重要特征。

AI输入内容

请把下面的JSON转成表格形式

{
  "星期一": ["语文", "数学", "英语"],
  "星期二": ["物理", "化学", "生物"],
  "星期三": ["历史", "地理", "政治"]
}

AI输出结果

星期 科目
星期一 语文、数学、英语
星期二 物理、化学、生物
星期三 历史、地理、政治

此时AI可自动识别表格中的字段结构，如"星期"和"科目"，实现对课程数据的分类、统计与扩展应用，例如生成课程表、安排提醒或分析学习频率。

📝 板块五：课后总结

（一）核心回顾

Markdown

AI理解人类需求的指令语言

JSON

AI与机器之间的数据语言

掌握两者，意味着能够完成从
"意图表达" 到 "结构执行"
的完整链路

（二）为什么AI更喜欢结构化

1

AI理解语义的方式不同

人类依靠语言语境，AI依靠数据结构。JSON让AI以更高精度理解和操作信息。

2

结构化让AI结果可控

非结构化文字仅"看起来正确"；结构化数据才能被"自动化使用"。

ChatGPT 输出JSON 自动生成PPT 自动发送邮件
3

从"对话"到"编排"

Markdown负责表达意图，JSON负责执行逻辑。这两种语言共同构成了AI应用的语义底座。
