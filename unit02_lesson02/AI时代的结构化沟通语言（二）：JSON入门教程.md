# AI时代的结构化沟通语言（二）：JSON入门教程

# 板块一：从Markdown到JSON

## （一）结构化语言的演进

上节课我们学过 **Markdown** —— 它是一种让文字更整齐、层次更清楚的写作方式。用Markdown，AI能更容易看懂我们想表达的重点和逻辑。

不过，当信息要被电脑程序**保存、传送或自动处理**时，只靠Markdown还不够。这时，AI需要一种**更标准、更结构化的表达方式**，让不同系统都能“听懂”同样的数据。这种语言就是 **JSON**。

## （二）Markdown与JSON的关系

Markdown和JSON其实就像是**AI沟通的两种语言**：

- **Markdown** 是写给“人”看的，讲究排版清楚、层次分明。
- **JSON** 是写给“机器”看的，讲究结构整齐、方便读取。

| 对比内容 | Markdown（写给人看） | JSON（写给机器看） |
| --- | --- | --- |
| 谁在用 | 人类读者 | AI或电脑程序 |
| 用来干什么 | 整理文字、做笔记、写文档 | 存放数据、传递信息 |
| 看起来 | 像普通文本 | 像一张“数据表” |
| 优点 | 好读、好看 | 结构清楚、AI能理解 |
| 举个例子 | `# 今天的计划`、`**重点复习**` | `{"日期": "2025-11-06", "任务": ["学习", "运动"]}` |

因此，可以将两者视为AI沟通体系的两端：

- **Markdown → 人告诉AI“要做什么”**
- **JSON → AI告诉系统“我做了什么”**

这正体现了AI时代“结构化语言”的核心价值——**让思维结构与数据结构保持一致**。

# 板块二：认识JSON——AI最懂的语言

## （一）JSON的定义与起源

**JSON（JavaScript Object Notation）** 是一种让电脑**看得懂、记得住的文字格式**。它最早是一个叫**Douglas Crockford**的人发明的。

你可以把它想成：电脑之间聊天用的“简单语言”。

现在，不论是 AI、网站，还是 手机App，它们在交换和保存信息时，都会用到这种格式。

## （二）JSON的核心特点

1. **轻巧好懂**
    
    JSON 就是普通的文字文件，不需要安装复杂软件。
    
    写法简单、结构清晰，人能看懂，电脑也能懂。
    
2. **有条理且有结构**
    
    JSON 就像一张**信息表格**，每条信息都有“名字”和“内容”两个部分。
    
    比如我们想记录一个学生的信息，可以这样写：
    
    ```json
    {
      "姓名": "小明",
      "年龄": 18,
      "成绩": 95
    }
    ```
    
    这里一共三行，每一行：
    
    - 左列是“名称”（比如 姓名、年龄、成绩）
    - 右列是“内容”（比如 小明、18、95）
    
    这样写的好处是：**清楚、有条理、电脑也能轻松理解。**
    
3. **到处都能用**
    
    不管你用什么语言编程，或者在哪个平台上运行（电脑、网页、手机），**JSON都能通用**。
    
4. **内容清楚明白**
    
    每条数据都有名字说明，不用猜。
    
    比如：`"`分数`": 95` 一眼就能看出是“分数是95”。
    
5. **用途广泛**
    
    从网站传数据、App设置、到AI模型的结果，**JSON几乎无处不在**。
    

## （三）AI为什么离不开JSON

在人工智能工作时，电脑需要不断**接收信息、处理信息、再输出结果**。如果每个系统说的“数据语言”都不一样，电脑之间就会听不懂、对不上。

这种数据格式（JSON）正好解决了这个问题：

- **输入结构化**：电脑能清楚知道每一项信息代表什么。
- **输出结构化**：别的系统拿到结果后，可以马上使用，不用再整理。

例如，当AI生成一篇新闻摘要时，如果以JSON格式输出：

```json
{
  "标题": "人工智能正在改变教育",
  "摘要": "AI帮助学生更高效学习，也让教学更有针对性。",
  "关键词": ["人工智能", "教育", "学习"]
}
```

这样，新闻网站就能：

- 自动把“标题”当作新闻题目，
- 把“摘要”放到正文里，
- 用“关键词”来做分类或搜索。

这体现了JSON在AI生态中的“连接功能”——**让AI结果变成电脑“可执行的数据”**。

# 板块三：JSON语法入门

### JSON的结构规则与数据类型

1. **对象（Object）**
    - 用花括号 `{}` 表示；
    - 内部由若干“键–值对”组成；
    - 键与值之间用冒号 `:` 连接；
    - 每组信息之间用逗号 `,` 隔开。
    
    **举个例子：**
    
    ```json
    {
      "名字": "小明",      // 用{}包起来的是对象
      "年龄": 20,          // “名字”和“年龄”是“键”，也就是信息的名称，后续会讲解
      "爱好": ["音乐", "运动", "游戏"]  // “值”可以是文字、数字、列表等，每组之间用 , 号隔开。
    }
    ```
    
2. **键（Key）**
    - 键是“信息的名字”，必须用双引号 `" "` 包起来。
    - 例如 `"名字"`、`"年龄"`。
3. **值（Value）**
    
    值是“具体的内容”，可以是：
    
    - 字符串：文字（如 `"小明"`）
    - 数字：数字（如 `20`）
    - 布尔值：是非判断（如 是`true` 或 非`false`）
    - 数组：列表，如 `["音乐","运动"]`
    - 对象：甚至是另一个对象（`{ }`）
    - 空值：表示“暂时没有内容”或“未知”（`null`）
    
    **举个例子：**
    
    ```jsx
    {
      "名字": "小明",                 // 字符串：文字内容，用双引号包起来
      "年龄": 20,                    // 数字：不用加引号
      "是否学生": true,              // 布尔值：true 表示“是”，false 表示“否”
      "爱好": ["音乐", "运动", "游戏"],  // 数组：一组内容，用 [ ] 包起来
      "家庭": {                      // 对象：里面又是一个小的“信息表”
        "父亲": "小王",
        "母亲": "小李"
      },
      "电话": null                   // 空值：目前还没有内容，用 null 表示
    }
    ```
    
4. **数组（Array）**
    - 用方括号 `[ ]` 包起来；
    - 表示一组内容，比如一串爱好、朋友名单等。
    
    **举个例子：**
    
    ```json
    "爱好": ["音乐", "运动", "游戏"]
    ```
    
    表示“爱好”这一项有三个内容。
    
5. **逗号规则**
    - 每组内容之间要用逗号 `,`；
    - 最后一组后面**不能加逗号**。
    
    **错误示例：**
    
    ```json
    {
      "名字": "小明",
      "年龄": 20,
    }
    ```
    
    **正确示例：**
    
    ```json
    {
      "名字": "小明",
      "年龄": 20
    }
    ```

# 板块四：AI结构化输出实践

结构化输出是AI实用化的重要特征。

**AI输入内容：**

> 请把下面的JSON转成表格形式
> 

```jsx
{
  "星期一": ["语文", "数学", "英语"],
  "星期二": ["物理", "化学", "生物"],
  "星期三": ["历史", "地理", "政治"]
}
```

**AI输出结果：**

| 星期 | 科目 |
| --- | --- |
| 星期一 | 语文、数学、英语 |
| 星期二 | 物理、化学、生物 |
| 星期三 | 历史、地理、政治 |

此时AI可自动识别表格中的字段结构，如“星期”和“科目”，实现对课程数据的分类、统计与扩展应用，例如生成课程表、安排提醒或分析学习频率。

# 板块五：课后总结

## （一）核心回顾

- **Markdown**：AI理解人类需求的指令语言；
- **JSON**：AI与机器之间的数据语言。

掌握两者，意味着能够完成从“意图表达”到“结构执行”的完整链路。

## （二）为什么AI更喜欢结构化

1. **AI理解语义的方式不同**
人类依靠语言语境，AI依靠数据结构。
JSON让AI以更高精度理解和操作信息。
2. **结构化让AI结果可控**
非结构化文字仅“看起来正确”；
结构化数据才能被“自动化使用”。
    
    例如：
    ChatGPT → 输出JSON → 自动生成PPT → 自动发送邮件。
    
3. **从“对话”到“编排”**
Markdown负责表达意图，JSON负责执行逻辑。
这两种语言共同构成了AI应用的语义底座。
