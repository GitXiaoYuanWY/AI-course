# 从0-1带你制作一个AI视频

# 板块一：课程概述

## 一、课程回顾

前两节课程分别介绍了 AI视频创作的基本概念与流程，以及 提示词与脚本、分镜、文案的编写技巧。学生已经掌握了如何使用提示词表达画面意图，如何通过结构化的脚本控制镜头表现，以及如何运用镜头语言、光影与氛围等要素让AI视频生成更加符合视觉叙事逻辑。

在上节课的课后作业中，学生尝试了编写分镜脚本并生成视频的过程。实际操作表明，虽然逐镜编写脚本能够获得较高的画面控制力，但这种方式往往存在以下问题：

1. 描述不精确 —— 如果提示词表述不清晰，AI模型会生成与预期不符的画面；
2. 操作繁琐 —— 每个镜头都需手动编写细节，如动作、角度、光线、景深等；
3. 主体不统一 —— 在多镜头组合中，不同分镜的角色或主体可能出现差异，影响成片一致性。

**因此，本课程将进入AI视频创作的高级阶段——批量化与系统化的生成控制。学习重点在于：**

- 利用参考图提升视觉一致性与内容精度；
- 使用提示词模板快速生成分镜与脚本；
- 通过角色系统维持主角造型与性格的统一。

## 二、学习目标

通过本节课程的学习，学生将能够：

1. 学会运用CGML（电影生成标记语言）描述视频结构与镜头逻辑；
2. 理解并熟练使用Sora2角色系统以维持角色一致性；

## 三、课程目录

- 使用提示词模板与图片快速生成AI视频
- Sora2角色创建与一致性控制

# 板块二：课程章节

## 一、使用提示词模板与图片快速生成AI视频

### （一）参考图与视觉输入的重要性

1. 参考图定义
    
    在AI视频生成中，“参考图”（Reference Image）是用于指导视觉风格与构图方向的核心输入之一。
    
    它相当于“视觉样本”，帮助模型理解场景、主体造型、色彩倾向等元素，使生成的画面更接近设计目标。在Sora2等AI视频平台中，参考图的输入形式通常为一张或多张图像文件。模型会分析其内容特征，并将其与提示词信息融合，生成具有更高一致性和视觉可信度的动态画面。
    
2. 参考图的功能包括：
    
    
    | 功能类别 | 说明 | 示例 |
    | --- | --- | --- |
    | **造型一致性** | 保证人物或动物在不同镜头中的外观保持稳定 | 同一角色在多个镜头中服装、发型不变 |
    | **风格参考** | 指导画面整体的色调、氛围或光线风格 | 使用阳光泳池照片确定色彩风格 |
    | **构图参考** | 帮助AI理解主体比例与背景位置 | 提供航拍照片以维持空间布局 |
    | **动作参考** | 提供运动姿态或关键帧方向 | 例如狗跳入水的瞬间照片 |

### （二）提示词模板的核心结构

1. CGML提示词定义
    
    AI视频生成的关键在于“语言如何转化为视觉”。为了实现精确控制，需要将自然语言提示词转化为可解析的结构化描述语言。
    
    “电影生成标记语言”（**Cinematic Generator Markup Language, CGML**）正是为此目的而设计的格式。它类似于电影分镜脚本的机器语言版本，通过规范化语法让AI理解镜头顺序、时长、动作、光影等要素。
    
2. CGML提示词：
    
    ```jsx
    ## 1. 角色与定位
    
    你是一名专业的**电影式AI视频生成提示词架构师**。你的核心职能是将用户模糊的创意想法，转化为结构严谨、描述精确、可直接用于驱动专业级AI视频生成模型的提示词。你输出的结果必须严格遵循名为 **“电影生成标记语言（Cinematic Generator Markup Language, CGML）”** 的标准化格式。
    
    ## 2. 核心工作流程
    
    你的工作遵循一个清晰的三步流程：
    
    1. **需求澄清**：主动与用户对话，挖掘并确认其创意中的关键细节，确保理解无歧义。
    2. **结构化构建**：将确认后的信息，系统地填充到标准的CGML框架中。
    3. **最终输出**：生成完整的CGML提示词，并可选择性地附上“导演阐述”，解释关键设计思路。
    
    ## 3. CGML输出格式规范
    
    你输出的所有提示词都必须完全遵循以下结构。每个镜头都是一个独立的模块，包含以下所有必填字段。
    
    [项目标题] — [场景描述] ## Structure (total [总时长]s; content [内容时长]s + [黑场时长]s blackout; mode: [模式]; tempo_factor: [节奏因子])
    
    - [ #1 [镜头时长]s ] ACTION:{描述场景内发生的具体动作} ; CAMERA:{描述摄像机的运动、角度和构图} — [[STATIC/DYNAMIC]]
    | SUBJECT:{描述主体（如角色）的外观、服装、装备及状态}
    | SCENE:{描述环境、背景及场地细节}
    | LIGHT:{描述灯光类型、方向、性质、主次光源}
    | GRADE:{描述色彩风格、对比度、饱和度、滤镜等后期处理}
    | CAM:{描述镜头焦距、景深、镜头类型及任何技术细节}
    | AUDIO:{描述环境音、音效、音乐氛围}
    - [ #2 [镜头时长]s ] ACTION:{...} ; CAMERA:{...} — [[STATIC/DYNAMIC]]
    | SUBJECT:{...}
    | SCENE:{...}
    | LIGHT:{...}
    | GRADE:{...}
    | CAM:{...}
    | AUDIO:{...}
    
    ... (后续镜头)
    
    ## 4. 用户交互协议
    
    在开始构建提示词前，你必须首先向用户收集或确认以下信息。如果用户已提供部分信息，则在此基础上询问缺失项。
    
    **启动语：**
    “您好，我是您的电影式AI视频生成提示词架构师。为了为您构建精准的场景，请告诉我以下信息（如已有想法，可直接告知）：
    
    1. **核心概念**：请用一两句话描述您想要的场景。
    2. **主角/主体**：场景中的核心主体是什么？请描述其关键特征（如外观、服装）。
    3. **关键动作与叙事**：在这个片段中，发生了什么？故事是如何推进的？
    4. **氛围与风格**：您希望呈现何种视觉风格和情绪基调？（例如：赛博朋克、史诗奇幻、忧郁、悬疑）
    5. **特别要求**：是否有任何必须包含的视觉元素、光线效果或声音设计？
    
    如果您对任何细节不确定，我可以提供建议。”
    
    ## 5. 输出规则与质量要求
    
    - **格式**：必须严格使用上述CGML格式，确保所有模块完整。
    - **语言**：必须使用专业、精准且富有画面感的词汇（例如：`volumetric lighting`（体积光）、`chiaroscuro`（明暗对比法）、`dolly in`（推近））。
    - **完整性**：每个镜头都必须包含 `ACTION`, `CAMERA`, `SUBJECT`, `SCENE`, `LIGHT`, `GRADE`, `CAM`, `AUDIO` 八个模块。
    - **技术规范**：必须为每个镜头指定确切的时长和类型（`STATIC` 或 `DYNAMIC`）。
    - **纯净性**：最终输出必须是纯文本，禁止使用Markdown标题、列表或其他格式，以确保与生成系统的兼容性。
    
    ```
    

### （三）案例：边牧犬的夏日泳池

1. 提供参考图
    
    在AI视频生成中，参考图的作用是让AI理解视觉风格与主体特征。
    
    在本案例中，选用一张**边境牧羊犬叼着橙色球的照片**作为参考图。
    
    ![1addeb49579d1a1fff72592b76ae5373.png](1addeb49579d1a1fff72592b76ae5373.png)
    
    这张图帮助AI确定：
    
    - 主体外观：黑白毛发、橙色球；
    - 场景色调：蓝色泳池、水花反光；
    - 光线特征：阳光直射、暖调高光。
    
    通过参考图，AI在生成视频时能够保持犬只造型、色彩与环境的一致性。
    
2. 输入视频需求
    
    输入的文本描述（即核心创意）如下：
    
    ```
    核心概念：一直叼着橙色球的边牧犬在泳池里玩水
    主角：边牧犬
    动作：追逐、嬉戏
    风格：活泼、快乐
    特别要求：有特写
    ```
    
    这一部分是视频生成的要求，帮助AI明确故事主题和镜头情绪基调。
    
3. 生成提示词（CGML格式）
    
    通过提示词模板系统自动生成以下结构化脚本：
    
    ```jsx
    [项目标题] — 边牧犬的夏日泳池 ## Structure (total 12s; content 12s + 0s blackout; mode: cinematic_realism; tempo_factor: 1.0)
    
    [ #1 4s ] ACTION:{一只叼着橙色球的边牧犬跃入泳池，水花飞溅，阳光下形成金色波光。} ; CAMERA:{航拍俯视镜头，缓慢推近，展示水面纹理与犬的动态轨迹。} — [DYNAMIC]
    | SUBJECT:{边境牧羊犬，黑白相间毛色，毛发被水打湿，嘴叼鲜艳橙色球，神态兴奋、专注。}
    | SCENE:{户外泳池，水面清澈，四周环绕棕榈与木质平台，背景模糊的夏日庭院。}
    | LIGHT:{自然光，强烈日照，阳光反射在水面形成高光闪烁。}
    | GRADE:{高饱和度色彩，明亮蓝绿调，色温偏暖，营造夏日清爽感。}
    | CAM:{无人机镜头，焦距约35mm，广角轻微畸变增强动感。}
    | AUDIO:{清脆水花声与犬的呼吸声，背景为轻快的夏日音乐节奏。}
    
    [ #2 5s ] ACTION:{边牧在水中扑腾、咬住橙色球并抬头甩动，水珠飞溅四散。} ; CAMERA:{从中景缓慢切换至低角度仰拍，随后短暂推镜至特写，捕捉水珠飞舞瞬间。} — [DYNAMIC]
    | SUBJECT:{毛发滴水的边牧，表情兴奋，橙球在嘴中轻微滚动，舌头微露。}
    | SCENE:{泳池中央区域，水波层叠，阳光穿透波纹。}
    | LIGHT:{逆光拍摄，阳光从后上方照射，形成金色边缘光。}
    | GRADE:{略带镜头炫光与柔焦效果，增强梦幻质感。}
    | CAM:{使用中长焦镜头（约85mm），浅景深突出主体，慢速快门捕捉水滴轨迹。}
    | AUDIO:{强化水声与犬吠回响，配以轻节奏鼓点背景。}
    
    [ #3 3s ] ACTION:{边牧叼球回游至池边，抬头望向镜头，眼神充满期待与快乐。} ; CAMERA:{手持式近景镜头稳定推进，定格在犬的脸部特写。} — [STATIC]
    | SUBJECT:{边牧面部特写，湿润毛发与橙球形成强烈对比，眼中倒映阳光。}
    | SCENE:{背景虚化，仅留浅蓝水面与微光反射。}
    | LIGHT:{正面柔光与微微闪动的反射光结合，突出毛发细节。}
    | GRADE:{高对比、暖调滤镜，强调阳光质感。}
    | CAM:{手持镜头，焦距50mm，浅景深对焦于犬眼与橙球。}
    | AUDIO:{环境声逐渐淡出，仅保留轻音乐尾声与呼吸声。}
    
    ```
    
    该结构化提示词确保每个镜头内容清晰、参数完整、语义精准，AI能直接据此生成高质量视频。
    
4. 输入到Sora**2**中
    
    将上述提示词粘贴至 Sora2平台的文本输入框：
    
    - 附加参考图；
    - 确认时长、分辨率与风格参数后点击生成。
    
    Sora系统会自动解析CGML格式文本，并根据参考图特征与提示词语义生成连续镜头画面。

## 二、Sora2角色创建与一致性控制

### （一）Sora2角色系统概述

1. 角色功能定义
    
    Sora2是基于文本的视频生成平台，其角色系统用于实现视觉与性格的一致性管理。在AI生成多镜头视频时，角色一致性是决定成片质量的关键因素。
    
    “角色”在Sora2中并非传统意义上的动画人物建模，而是由用户上传的参考视频或片段提取出的特征集合。
    
    ![image.png](image%201.png)
    
    角色系统的主要作用包括：
    
    - 视觉一致性：
        
        确保角色在不同镜头或项目中保持相同面部特征与服饰风格。
        
    - 动作一致性：
        
        AI可根据角色视频学习其动作节奏，使其行为更符合人设。
        
    - 性格呈现：
        
        通过描述字段，系统可在语义层面保持性格逻辑一致，如“冷静沉稳”“活泼外向”。
        
    - 制作效率提升：
        
        角色模板可复用于多个视频项目，减少重复定义。
        
2. 角色功能应用场景
    
    Sora2允许在同一项目中同时调用多个角色（最多3个）。系统会根据提示词内容与镜头结构自动分配出镜角色。此功能特别适合：
    
    - 对话类视频（如访谈、剧情）；
    - 多人物场景（如团队活动、舞蹈片段）。
    
    若需确保角色不混淆，可在提示词中明确标注各自的行动与视角。
    

### （二）角色创建流程详解

1. **视频素材准备**
    - 输入文件必须为视频格式（≥3秒），图片不被系统接受；
    - 推荐使用动画片段或实拍片段中的清晰镜头；
    - 禁止使用360°环绕视频，以防系统识别失败。
2. **上传与命名**
    - 打开主页，点击创建角色，上传你准备的视频素材
        
        ![image.png](image%202.png)
        
        ![image.png](image%203.png)
        
        ![image.png](image%204.png)
        
        ![image.png](image%205.png)
        
    - 成功上传后进入角色描述页面；
    - 角色命名可自定义，避免使用已有知名名称，以防冲突。
3. **角色描述填写**
    - 默认描述即可满足大多数场景；
    - 若需增强一致性，可补充性格、语气或行为特征。
4. **保存与调用**
    - 保存后角色即创建完成；
    - 可通过“@角色名”方式直接调用。

### （三）常见问题与解决方法

| 问题类型 | 可能原因 | 解决方法 |
| --- | --- | --- |
| 角色上传失败 | 视频太短或360°视频 | 使用≥3秒的固定视角片段 |
| 角色生成崩坏 | 3D素材比例不符 | 采用实拍或2D片段 |
| 角色调用失败 | 名称错误或缓存未刷新 | 手动输入名称并刷新页面 |
| 图像失真 | 光线或背景复杂 | 使用干净背景、均匀光线素材 |

角色系统的有效运用，可以显著提升AI视频的连续性与专业度，尤其在叙事型作品中尤为重要。

# 板块三：课后实战任务

## 一、任务名称

> 创建一个喜欢的角色并生成一版提示词，使用CGML提示词模板生成完整AI视频。
> 

## 二、操作步骤与要求

1. 角色创建阶段
    - 录制或选取一段清晰的3秒以上视频；
    - 上传至Sora2并命名角色；
    - 填写角色描述，保持默认即可。
2. 提示词生成阶段
    - 使用CGML模板输入创意概念；
    - 明确主体、动作、氛围与风格；
    - 若需统一风格，可引用前述光影类型。
3. 生成与验证阶段
    - 将提示词输入Sora2；
    - 调整镜头顺序与时长以获得最佳节奏；
    - 检查角色一致性与画面细节。
4. 成果展示阶段
    - 保存生成视频；
    - 对比角色与脚本一致性效果；
    - 分析光影与氛围是否匹配预期。

## 三、评估要点

| 评估维度 | 内容说明 |
| --- | --- |
| **脚本完整性** | 提示词结构符合CGML标准，字段齐全 |
| **角色一致性** | 同一角色在多镜头中形象统一 |
| **光影与氛围** | 画面风格与主题匹配 |
| **动作自然度** | 镜头衔接平滑、动作连贯 |
| **创意表达** | 场景具备故事感或视觉冲击力 |
