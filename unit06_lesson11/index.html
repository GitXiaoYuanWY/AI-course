📝 一、在模板中替换文本内容

（一）基本原则

💡 核心提示：

修改内容时，只改变标签之间的文字，保持HTML结构和属性不变。就像在填空题中填写答案，不要修改题目本身！

（二）About 模块示例

HTML 结构示例 
<section id="about" class="section section--about">
  <div class="section__inner">
    <h2>关于我</h2>
    <p>我是一名新媒体专业毕业生，专注于活动策划与执行、新媒体运营等领域。</p>
  </div>
</section>
可修改部分
  • "关于我" 可根据需要调整
  • 段落内容替换为个人简介
  • 建议 100-150 字
保持不变部分
  • id="about" 不要修改
  • class 属性保持原样
  • HTML 标签结构不变

（三）Skills 模块示例

HTML 结构示例 
<section id="skills" class="section section--skills">
  <h2>技能护照</h2>
  <ul class="skills-list">
    <li>活动策划与执行</li>
    <li>新媒体运营</li>
    <li>品牌推广</li>
    <li>项目管理</li>
    <li>内容创作</li>
    <li>数据分析</li>
  </ul>
</section>
📌 建议：

保持 3-6 条技能，格式统一为简短名词短语

🖼️ 二、替换头像与项目图片

（一）图片存放位置与命名建议

public/ avatar.jpg ← 头像图片 project1.jpg ← 项目1封面 project2.jpg ← 项目2封面 project3.jpg ← 项目3封面 ...

文件命名规范

✓ 推荐做法
  • 使用英文字母、数字、短划线
  • 简短且可辨识：avatar.jpg
  • 避免空格和特殊字符
  • 统一小写字母
✗ 不推荐做法
  • 使用中文：我的头像.jpg
  • 包含空格：my avatar.jpg
  • 大小写混用：Avatar.JPG
  • 特殊字符：avatar@2x.jpg

（二）更换头像示例

代码示例 
<img src="public/avatar.jpg" alt="个人头像" class="avatar">
⚠️ 重要提醒：

GitHub Pages 区分大小写，Avatar.jpg 与 avatar.jpg 被视为不同文件！确保路径与文件名完全一致。

🎨 六、调整主题颜色与基础风格

颜色选择建议

主色选择

  • 选择与个人风格相关的颜色
  • 确保与背景有足够对比度
  • 保证文字可读性

辅色选择

  • 与主色协调（邻近色或互补色）
  • 避免过于鲜艳
  • 考虑整体视觉平衡

🐛 七、修改中常见错误与简单排查

图片不显示

症状：图片位置显示为空白或破损图标
常见原因：
  • 路径错误（相对路径写错）
  • 文件名大小写不匹配
  • 文件未上传到正确位置
  • 图片格式不支持
AI 提示词： "xxx位置的图片未显示，我已确认文件上传在xxx位置。请检查路径与文件名大小写是否一致，并进行修正。"

样式不生效

症状：修改 CSS 后页面样式无变化
常见原因：
  • CSS 语法错误（缺少分号、括号不匹配）
  • 选择器不匹配
  • 浏览器缓存未清除
  • 样式优先级被覆盖
AI 提示词： "我修改了 CSS 变量但没有生效，请检查 :root 中变量是否正确被引用、选择器是否匹配。"

布局错乱

症状：元素重叠、错位或宽高异常
常见原因：
  • HTML 标签未正确闭合
  • 意外删除关键结构
  • CSS 样式冲突
  • Flexbox/Grid 属性错误
AI 提示词： "xxx位置的项目卡片布局错乱，可能标签闭合出错。请检查 HTML 结构。"

链接无法点击

症状：按钮或链接点击无反应
常见原因：
  • href 属性错误或缺失
  • JavaScript 函数错误
  • 元素被其他层遮挡（z-index）
  • pointer-events 被禁用
AI 提示词： "xxx位置的按钮点击无响应，请检查 JS 控制函数或 href 是否正确。"

预防错误的建议

🔹

小步修改

每次只修改一个小区域，保存并测试后再继续

💾

使用版本控制

定期提交代码到 Git，添加有意义的提交信息

📋

保留备份

重大修改前，复制原始文件作为备份

🧪

逐步测试

本地充分测试后再部署，在多个浏览器中测试

🚀 八、GitHub部署网页

Git 命令流程

1

添加修改的文件

git add .

添加所有修改的文件到暂存区

2

提交修改

git commit -m "更新个人信息和项目"

提交修改并添加描述信息

3

推送到远程仓库

git push origin main

将本地修改推送到 GitHub

4

等待自动部署

GitHub Pages 会自动检测更新并部署（通常需要1-3分钟）

✓ 检查清单：
  • 访问个人作品集链接，确认所有修改已正确显示
  • 检查图片是否正常加载
  • 测试导航链接是否正常工作
  • 测试项目卡片交互功能
  • 在不同浏览器中测试兼容性
