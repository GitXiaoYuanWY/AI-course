🔍 一、网页部署前需要检查什么？

视觉一致性

  • 字体体系统一（标题/正文字号级差、字重一致）
  • 主辅色与中性色统一（按钮、链接、标签颜色与状态一致）
  • 间距刻度统一（如 8/16/24/32）
  • 卡片圆角、阴影风格一致

内容准确性

  • 个人简介 100–150 字，信息密度明确
  • 项目卡片标题 ≤12 字
  • 项目摘要 40–60 字
  • 技能标签 3–6 个
  • 链接/按钮文案清晰明确

结构完整性

  • 导航跳转正常（锚点 id 是否对应）
  • 模块顺序符合预期
  • 语义化标签正确
  • Hero → About → Projects → Contact → Footer

移动端基础适配

  • viewport 元标签已添加
  • 断点下网格列数与留白合理
  • 图片不溢出（max-width:100%）
  • 文字大小适合触屏操作

📋 详细检查清单

✓
HTML元信息：<title> 与 <meta name="description"> 是否编写
✓
网站图标：favicon 是否配置
✓
页面顺序：About → Skills → Projects → Contact 是否按规划呈现
✓
链接可用性：所有按钮/链接可用（外链新开窗 target="_blank" rel="noopener"）
✓
图片资源：路径正确、大小适中（建议 120–200KB 级别）
✓
内容完整性：无错别字、空白模块、占位文案未替换
✓
颜色对比：标题/正文与背景对比度≥4.5:1
✓
代码组织：使用CSS变量/统一间距刻度，无重复样式

💻 三、GitHub Pages 部署实操

从零开始，手把手教你将网页部署到GitHub Pages

步骤 1

注册 GitHub 账户

1

访问 github.com

2

点击右上角"Sign up"创建账号（邮箱 + 密码）

3

验证邮箱后登录，进入主页

提示：用户名建议使用专业化的名称，因为它会出现在你的网页URL中

步骤 2

创建新仓库（Repository）

1

点击右上角 "+" → New repository

2

填写仓库名称（建议：my-portfolio）

3

选择 Public（必须公开，否则无法部署）

4

点击 "Create repository" 创建仓库

💡 什么是仓库？

仓库相当于一个"网盘文件夹"，部署就从这里开始。它存储你的所有网页文件，并提供版本管理功能。

步骤 3

上传网页文件

1. 准备文件

在部署前，需要把网页相关的所有文件整理成一个清晰的文件夹结构：

my-portfolio/
index.html
assets/
css/
style.css
js/
script.js
img/
project1.jpg
avatar.png
favicon.ico

2. 上传步骤

1

进入仓库页面

2

点击 "Add file" → "Upload files"

3

拖入所有网页文件与文件夹（保持原结构）

4

下拉页面，点击 "Commit changes"

步骤 4

启用 GitHub Pages

1. 进入设置

1

打开仓库

2

点击右上角 "Settings"

2. 启用 Pages

1

左侧选择 "Pages"

2

在 Source 中选择：Branch: main

3

Folder: / (root)

4

点击 "Save"

🎉 部署成功！

稍等数十秒，页面会刷新并显示：

Your site is published at:
https://你的用户名.github.io/仓库名/

这就是你的网页访问链接！可以发送给HR、放在简历上、分享给他人随时访问

⚠️ 四、部署常见问题与解决方式

访问链接显示 404 / Not Found

常见原因：
  • 根目录缺少 index.html
  • 部署目录不是 / (root)
  • Pages 尚未构建完成
解决方式：
  • 将首页改名为 index.html
  • Pages 设置：Branch = main / Folder = / (root)
  • 等待构建完成后刷新链接

图片不显示

常见原因：
  • 路径大小写不一致
  • 图片未上传
  • 使用了绝对路径 /assets/...
解决方式：
  • 将目录和文件名统一小写
  • 使用相对路径如 ./assets/...
  • 检查仓库中的真实路径

CSS / JS 不生效

常见原因：
  • 路径错误
  • 文件名拼写错误
  • 浏览器缓存
解决方式：
  • 校对文件名是否完全一致
  • 强制刷新（Ctrl+F5）
  • 引入 ?v=1 破缓存

移动端错位 / 布局乱

常见原因：
  • 缺少 viewport 标签
  • 图片宽度未限制
  • Flex/Grid 断点未设置
解决方式：
  • 添加 viewport 元标签
  • 设置 img{max-width:100%}
  • 基础断点写法：@media (max-width:768px)

🔄 五、更新与修改网页文件

部署完成后，你可以随时更新网页内容。GitHub Pages 会在检测到新提交时自动重新构建。

方法1：直接替换文件

1. 点击 "Add file" → "Upload files"

2. 拖入新版本文件

3. GitHub 自动识别并覆盖旧版本

4. 点击 "Commit changes"

适用：更新 HTML、CSS、JS、图片等

方法2：在线编辑文件

1. 打开文件后，点击铅笔图标（Edit）

2. 在 GitHub 编辑器中修改内容

3. 下拉选择 "Commit changes"

适用：修改文案、调整 CSS 颜色值、修正路径等

更新后多久能看到变化？

GitHub Pages 一般需要 20秒～2分钟 自动构建

  • 刷新浏览器查看
  • 如未更新，可尝试清理缓存（Ctrl + Shift + R）
  • 若3分钟仍未更新，检查是否提交到 main 分支

💡 更新目录结构（添加新文件夹）

如果需要新增文件夹（如 assets/img）：

  1. 点击 "Add file" → "Create new file"
  2. 在名称处输入：assets/img/占位.txt
  3. Commit 后，即可上传真实图片文件
  4. 删除 占位.txt（可选）

🤖 六、AI 辅助网页优化

使用AI可以帮助你进行代码级优化，提升网页质量和可维护性

清理未用代码

让AI帮你识别并删除未使用的CSS样式和JS函数

提示词示例： "帮我检查这个CSS文件，找出所有未在HTML中使用的样式，并删除它们"

统一CSS变量

将重复的颜色、字号、间距值提取为CSS变量

提示词示例： "将这个CSS文件中重复出现的颜色值统一为CSS变量"

补充注释

为关键代码段添加清晰的注释，提高可读性

提示词示例： "为这段JavaScript代码添加清晰的注释，说明每个函数的作用"

合并重复样式

识别并合并重复的CSS规则，减少代码冗余

提示词示例： "帮我找出所有重复的CSS样式，并将它们合并优化"
