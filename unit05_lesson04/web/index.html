🔑 一、Suno 登录与账号基础

1. 访问与登录方式

Suno 是基于 Web 的 AI 音乐生成平台，服务器部署与运营环境位于海外。其服务面向全球用户，因此在部分地区访问时可能需要符合当地的网络环境条件。

登录方式主要包含以下几类：

  • 使用 Apple 账号登录
  • 使用 Discord 账号登录
  • 使用 Google 账号登录
  • 使用 Microsoft 账号登录
  • 中国大陆手机号不可用 → 建议用邮箱账号登录

2. 常见登录问题与解决策略

在实际使用中，登录环节经常出现的一类问题是"能打开首页但无法成功登录"。这种现象一般说明基础网络连接尚可，但登录请求在跨境传输或认证过程中被拦截或中断。

常见问题类型：

  • 点击登录按钮后页面长期无响应或返回错误码
  • 第三方授权页面无法跳转或无法正确回调
  • 登录后立即被登出，无法保持会话状态

解决策略：

网络调整

调整网络工具为全局模式，使浏览器与系统层面所有请求都走同一条出口

浏览器优化

尝试更换浏览器或清理缓存，排除本地 Cookie、缓存数据导致的异常状态

账号切换

使用不同的第三方账号尝试登录，有时具体授权渠道的稳定性存在差异

🏠 二、Suno 主页认识

1. 主页的主要区域

  • 顶部歌单/封面区：热门歌单 + 播放量、点赞、作者信息
  • 中部三块：
    • 为你推荐的歌曲
    • 推荐的创作者
    • 排行榜（可切语言/周榜/月榜）
  • "历史最热"区：几十万/几百万播放量作品
  • 热门人物、为你推荐创作者
  • New Song：刚刚公开的新歌
  • 按风格分类的歌单入口：流行、乡村、金属、民谣、摇滚等
  • 继续收听：相当于播放历史列表
  • 官方 Snowball 展示区：历史顶级播放作品

2. 主页的教学用途

Suno 主页不仅是一个被动接收推荐内容的空间，更是一个主动学习的平台工具。

1

提示词观察与对照

大部分公开作品会在详情区域展示创作者使用的提示词，包括风格描述、结构标签、情绪词汇等。通过对比提示词与实际听感，可以逐渐建立"文本描述与声音效果之间的对应关系"。

例如：某首作品的提示词中包含"warm nostalgic pop ballad, 95 BPM, soft piano and strings, male vocals"

在聆听时可以留意：

  • 温暖与怀旧感是如何通过和声与音色呈现的
  • 95 BPM 的节奏与常见流行歌曲相比处于何种速度区间
  • 软音色钢琴与弦乐如何在伴奏中分配角色
  • 男声演唱的音色位置与混响处理方式是否符合提示词描述
2

结构模仿与混合风格尝试

在观察多个作品的基础上，可以发现提示词在结构与风格组合上的共性模式，例如：

  • "[情绪] + [风格] + [年代] + [人声类型] + [编配要素]"
  • "[场景描述] + [氛围词] + [节奏强度]"

这些模式可以在后续创作中被改写、组合或扩展，用于新的音乐任务。

3

构建个人学习型播放列表

通过对主页推荐、排行榜、热门人物与历史最热的综合浏览，可以将具有代表性的作品加入收藏或记录清单，形成面向学习目的的播放列表。

在持续聆听与对比中，创作者可以逐渐形成对不同风格标签、提示词组合的"听觉印象"，这对于后续独立编写提示词与控制作品走向具有长期价值。

3. 简单模式核心点

💡 便利技巧：使用"沉浸式翻译"浏览器插件或其他翻译工具，在输入框内直接将中文转换为英文描述。连续敲击三次空格可以自动把对话框中的中文描述翻译成英文。

🎛️ 四、自定义模式（Custom）入门

1. 自定义模式入口与结构

在创作页面中，切换到 "Custom" 即可进入自定义模式。该模式通常包含几个关键区域：

  • 歌曲标题（Title）：用于标记作品名称，方便在多版本中进行识别与管理
  • 歌词输入区：可以输入完整的歌词文本，也可以使用平台的自动写词功能生成初稿
  • 音乐风格（Style）输入区域：通过标签组合描述总体风格，比如流派、情绪、节奏感、人声类型等
  • Instrumental 开关：控制是否保留人声部分，与简单模式作用一致
  • 更多选项（More Options）：通常包含排除风格元素、角色选择、语言偏好等高级设置

自定义模式将"文本提示"与"音乐结果"之间的映射关系显式化，创作者可以在一次次生成与修改中建立对各参数的直觉。

2. 写歌词的两种方式

1

手动编写歌词

手写歌词适用于已经明确主题、故事线和用词风格的创作情境。创作者可以生成多段 Verse、Chorus 与 Bridge，结合前期学习的结构标签来组织文本。

优势：叙事与表达的可控性较强，可以精确传达某一故事、人物关系、情绪变化或价值主张。

2

使用内置大模型自动写词

在歌词框中输入一段自然语言描述，例如"写一首关于雨夜重新振作的中文歌词"，系统会调用内置语言模型，生成与描述相匹配的歌词文本。

特点：自动写词通常包含多个段落、押韵结构与情绪递进，能够快速提供一份可用的歌词草稿。

💡 推荐工作流

先利用自动写词功能生成初稿，然后通过手动修改强化关键句、调整意象与节奏，使歌词既拥有模型的快速构思能力，又保留创作者的个人审美与经验判断。

3. 局部改词：By line 功能

选中需要修改的段落

比如"更痛一点""更温柔一点"等描述

4. 音乐风格标签与排除标签

在 Style 中指定风格

  • 流派：pop, R&B, acoustic, cinematic
  • 人声：male vocal / female vocal / duet
  • 年代：90s, 80s retro
  • 情绪：melancholic, uplifting, dark

在「排除」框中

  • 写入不想要的元素：
  • drums（鼓）
  • distorted guitar（失真吉他）
  • EDM 等

⚠️ 重要提示

这些标签提高"生成概率"，不是 100% 强制约束，会有 10% 左右"抽到意外"的情况（类似抽卡）。

➕ 五、续写功能

1. 续写功能的用途

对已有歌曲在某个时间点之后继续往后写：

  • 让歌曲变长
  • 改掉后半段的人声/歌词/风格（比如把后半段女声改成男声）

2. 操作思路（讲解 + 演示）

1

进入续写模式

找到要续写的歌 → 点击三个点 → Create → Extend

2

选择保留时间点

选择保留到哪一秒（例如 00:27）

  • 可手动输入时间
  • 注意是英文分号
3

删除已唱歌词

把已经唱完的歌词段落从歌词框中删除，避免"又唱一遍第一段"

4

填写新歌词

填写想要的新歌词/描写 → 生成新的 Part 2

5

合并完整歌曲

满意后 → 在 Part 2 上点击三个点 → Create → Get Both Song

  • 合并为完整一首歌
  • 这步不消耗积分

📝 补充规则说明

  • 单次续写最长约2分钟
  • 可对 Part 2 再续写，理论上可生成十几分钟长歌

✂️ 六、局部编辑

1. 功能定位与适用情况

局部编辑功能用于只修改歌曲中的某一小段，而保持其余部分不变。与续写功能相比，局部编辑更像是对现有作品的一种"局部重绘"：

  • 当整首歌整体风格满意，仅对某一段的旋律、人声性别或歌词不满意时，可以使用局部编辑
  • 当希望在某一部分尝试多种不同版本，如不同情绪强度、不同语言或不同和声结构时，也可通过局部编辑进行快速试验

这种细粒度修改方式在项目制作中非常实用，能够避免从头生成整首歌的重复工作。

2. 局部编辑的进入方式

通常可以通过两种方式进入局部编辑界面：

方式一

在歌曲详情或播放区域直接点击"Edit"按钮

方式二

通过歌曲卡片上的"三个点"菜单，依次选择"Song Edit → Repeat Session"等选项进入

进入局部编辑后，界面通常呈现为带有分段标记的波形或区块列表，每一段对应歌曲中的一个结构单元，如 Verse 1、Verse 2、Chorus 等。

3. 局部编辑的核心操作逻辑

1

选择需要修改的区段

在分段视图中，选择目标区段（例如 Verse 1）。有些系统会自动标注区段名称，提示对应的是主歌、副歌或桥段。

也有部分界面允许手动设置时间区间，通过拖动起始与结束节点来划定需要编辑的范围。

2

调整风格与人声设置

在选定区段后，可以在风格栏中输入新的标签，例如添加 female vocal，将原本的男声主唱段改为女声。

也可以改变情绪标签或配器密度，如从清淡编制改为更加厚重的摇滚伴奏。生成结果将仅在该区段生效，其他段落保持原样。

3

修改局部歌词内容

若某一段歌词内容需要修改，可以在歌词文本中找到对应句子，将其替换为新文本。在某些版本中，可以取消"持续时间"限制，使系统更自由地在时值上适配新歌词。

修改完成后，点击生成，系统将在该区段内重新演唱新歌词，同时尝试保持与前后段落在音色与混音上的连续性。

4

应用与对比

当新版本生成完成后，一般会提供试听功能。若结果满意，可以点击"Apply"或类似按钮，将该版本应用到整首歌曲中，替换原有区段。

在应用前，保留原版本与新版本的对比是一个重要步骤，有助于检验局部修改是否提升了整体表现力。

5

与创作页面联动

局部编辑完成并应用后，返回创建页面或作品详情，可以看到整体歌曲已更新为新的组合版本。此时可进一步通过续写或角色功能进行后续加工。

👤 七、角色功能介绍

1. 角色（Person）是什么？

角色功能可以理解为在 Suno 中为某一类"声音形象"建立一个可复用的模版。该模版往往包含下列特征的组合：

  • 固定或相对稳定的演唱音色与发声习惯
  • 相似的旋律构造与节奏偏好
  • 一致或相近的混响与空间感设置
  • 某种统一的风格倾向，如"温柔女声抒情风""低沉男声城市流行"等

💡 角色功能的作用

  • 对同一虚拟歌手进行多首歌曲的系列创作
  • 在保持音色与旋律特点的前提下，通过更换歌词与风格标签实现内容多样化
  • 为项目建立统一的声音 IP，例如游戏角色主题、虚拟主播专辑等

2. 创建角色的流程

1

选择基础歌曲

选中一首满意的歌 → 三个点 → Create → Make Personal

2

填写角色信息

  • 角色名字（如"爱的拼图·女生"）
  • 封面图（可选）
  • 描述文案（可选）
  • 是否公开给其他人使用
3

保存角色

保存后：系统会自动给这首歌勾选该角色

3. 使用角色创作

1

选择角色

在新歌的"更多选项"里选择 Add Personal → 选中刚创建的角色

2

编辑歌词并生成

再编辑歌词 → 生成

✨ 效果说明

  • 保留原有音色、旋律骨架
  • 只做细微内容变体（如歌词变化）
