🏗️ 二、网页是怎么工作的？

网页就像一栋房子，由三层组成：

HTML 结构层
"有什么" - 房子的骨架
协作
CSS 样式层
"长什么样" - 房子的装修
协作
JavaScript 行为层
"能做什么" - 房子的电路系统

HTML - 内容

定义页面有什么：标题、图片、按钮等

CSS - 外观

控制页面长什么样：颜色、大小、位置等

JavaScript - 交互

实现页面能做什么：点击、计算、更新等

理解这三层

就像盖房子：
• 先搭骨架（HTML）- 决定有几个房间
• 再装修（CSS）- 决定墙面颜色、家具摆放
• 最后接电路（JavaScript）- 决定开关灯、调温度

三层分工明确，互相配合，才能做出完美的网页！

🎨 四、第二层：CSS - 颜色和布局从哪来

为什么页面这么漂亮？秘密就在CSS里！ 

.price {
  color: #ff3b30;      /* 红色价格 */
  font-size: 32px;     /* 大字号 */
  font-weight: bold;   /* 加粗 */
}

.product-info {
  display: flex;       /* 横向排列 */
  gap: 40px;          /* 间距40px */
  padding: 60px;      /* 内边距 */
}
CSS的三大作用

控制颜色

文字颜色、背景颜色

控制大小

字体大小、元素宽高

控制布局

元素位置、排列方式

💡 CSS变量：一处修改，处处生效

:root {
  --primary-color: #0a84ff;  /* 定义品牌色 */
}

.button {
  background: var(--primary-color);  /* 使用品牌色 */
}

.link {
  color: var(--primary-color);       /* 使用品牌色 */
}

改一次 --primary-color，所有用到它的地方都会变！

📊 完整示例：价格动态计算的过程

步骤1
用户点击"皮革表带"选项
<li data-price="100">皮革表带</li>

HTML中存储了价格数据

步骤2
JavaScript获取data-price属性
const bandPrice = this.dataset.price; // "100"

从HTML元素中读取价格

步骤3
计算总价
const basePrice = 6499;
const total = basePrice + Number(bandPrice);
// 6499 + 300 = 6799

基础价 + 表带价 = 总价

步骤4
更新页面显示
priceDisplay.textContent = '¥' + total;
// 页面显示：¥6799

用户立即看到价格变化！

关键理解

这个过程展示了JavaScript如何让网页"动起来"：

  • HTML存储数据（data-price）
  • CSS美化样式（.active高亮）
  • JavaScript处理逻辑（读取、计算、更新）

三层结构完美协作！

⚠️ 六、AI也会犯错！

AI很强大，但不是完美的！
学会找出问题是重要技能。

案例：学会识别AI代码中的漏洞

不要盲目相信AI生成的代码！让我们来找找问题：

❌ AI生成的代码（有漏洞）
function validateForm() {
  const name = document.querySelector('#name').value;
  if (name.length > 0) {
    return true;
  }
}
问题清单：
  • ❌ 未使用 trim() 去除空格
  • ❌ 没有返回 false
  • ❌ 缺少错误提示
  • ❌ 只输入空格也能通过验证
✅ 改进后的代码
function validateForm() {
  const name = document.querySelector('#name').value.trim();
  if (name.length > 0) {
    return true;
  }
  alert('请输入姓名！');
  return false;
}
改进点：
  • ✅ 使用 trim() 去除首尾空格
  • ✅ 明确返回 false
  • ✅ 添加友好的错误提示
  • ✅ 逻辑更完整严谨
思考练习

这段代码还可以怎么改进？

  • 可以增加邮箱格式验证吗？
  • 可以检查姓名长度上限吗？
  • 可以用更友好的方式显示错误（而不是alert）吗？
重要提醒

AI生成的代码可能存在：
• 逻辑漏洞（如上面的例子）
• 安全问题（XSS、SQL注入等）
• 性能问题（循环嵌套过深）
• 兼容性问题（某些浏览器不支持）

永远保持批判性思维，理解每一行代码！

🔄 七、AI的多样性

同样的问题，AI可能给出不同的答案！
这既是优势，也需要你做选择。

多次生成的代码对比

使用同样的提示词，AI可能生成不同实现方式：

第一次生成

// 使用内联事件
<button onclick="buyNow()">
  购买
</button>

第二次生成

// 使用addEventListener
const btn = document.querySelector('#buyBtn');
btn.addEventListener('click', buyNow);

实验：同一提示词的不同生成结果

提示词："请生成一个智能手表的产品下单页"
对比项 第一次生成 第二次生成 评价
HTML结构 使用 <div> 标签 使用 <section> 标签 第二次更语义化 ✅
CSS布局 Grid布局 Flex布局 都合理，各有优势
事件绑定 内联 onclick addEventListener 第二次更符合分离原则 ✅
表单验证 有完整验证 无验证 第一次更完善 ✅
CSS变量 未使用 使用了 :root 第二次更易维护 ✅
代码注释 详细注释 较少注释 第一次更易理解 ✅
深入讨论
  1. 为什么同样的提示词会生成不同的代码？
    AI的生成具有随机性，每次会从多种实现方式中选择一种。
  2. AI是否有"记忆"？
    AI 没有长期记忆，每次生成都只依赖当前提供的提示，而不会自动引用之前的输出。
  3. 哪个版本更符合最佳实践？
    可以取各自优点：语义化标签 + addEventListener + 完整验证 + CSS变量。
  4. 如何获得更好的结果？
    改进提示词，明确要求：使用语义化标签、addEventListener、CSS变量等。
关键总结

学会阅读和修正AI代码是一项重要技能。我们要：

  • ✅ 理解代码的运行逻辑
  • ✅ 识别不合理的实现方式
  • ✅ 主动优化和改进代码
  • ✅ 保持批判性思维
