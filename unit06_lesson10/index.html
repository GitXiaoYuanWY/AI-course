📂 一、统一模板说明与文件结构简介

(一)本节使用的模板类型

实战版模板文件(个人版模板)

  • 用于后续实际修改和个性化
  • 页面是一个完整的作品集示例页面,没有额外提示文字
  • 代码中保留必要的注释(comment),说明哪些结构建议保留

教学版模板文件(带标注模板)

  • 用于直播讲解和结构解析
  • 页面中额外出现一些提示文字
  • 代码中注释更加详细,方便对照浏览器画面与源码

(二)文件结构示意(统一规范)

本节模板采用简化且统一的文件结构,方便后续部署与维护:

index.html ← 网页入口文件(首页)
styles.css ← 样式文件(页面外观)
main.js ← 行为脚本(基础交互)
assets/ ← 静态资源总文件夹
img/ ← 图片资源(头像、项目封面等)
icons/ ← 图标资源(社交媒体图标)

💡 重点:先看懂这些文件分别负责什么,不要求记住全部细节

🔍 三、在Trae中认识HTML结构骨架

(一)在Trae中打开模板与预览

  1. 打开Trae,进入本节课使用的作品集项目
  2. 在左侧文件列表中点击 index.html
  3. 确认右侧有页面预览,方便一边看画面一边对照代码

(二)浏览器画面与section的对应关系

作品集网页
Header 导航
Hero 展示区
About 关于我
Skills 技能
Projects 项目
Contact 联系
对应
<header>...</header>
<section id="hero">...</section>
<section id="about">...</section>
<section id="skills">...</section>
<section id="projects">...</section>
<section id="contact">...</section>

💡 关键要点:

  • ✅ 带有 id="hero" / "about" / "skills" / "projects" / "contact" 的 <section> 是对应的模块外层容器
  • ✅ 这些带 id 的 <section> 共同构成页面的结构骨架,后续不建议改名或删除

(三)"骨架"与"内容"的简单区分

结构骨架(建议不要随意改动)

  • <header> / <main> / <footer> 整体框架
  • 每个模块的 <section> 标签及其 id
  • 主要容器的层级关系(大致的嵌套结构)

可修改内容(下一节重点修改)

  • 文本节点(标题、段落、列表项)
  • 图片路径与替换图
  • 项目卡片的数量(以整块 <article> 为单位)

💡 本节只需认识:

想改"哪个区域的内容",就在Trae中先找到对应的 <section id="…">,后续只改内部文字和图片,不去改外层结构

🎨 四、在Trae中找到样式入口与主题控制

(一)定位主要样式文件与变量区域

  1. 在Trae左侧文件列表中找到与页面整体样式相关的CSS文件
  2. 打开主样式文件,在文件顶部或前半部分找到 :root { ... } 这一段
  3. 简要说明::root 中通常集中放置全局样式变量,用来控制主题颜色、背景色、文字色、圆角、阴影等

(二)识别主题相关的CSS变量

:root { ... }
--color-primary: #3498db
主色
--color-bg: #ffffff
页面背景色
--color-text: #333333
正文字体颜色
--radius-card: 12px
卡片圆角
--shadow-card: 0 4px 12px rgba(0,0,0,0.1)
卡片阴影
影响整个页面
按钮背景色使用 var(--color-primary)
卡片圆角使用 var(--radius-card)
卡片阴影使用 var(--shadow-card)

💡 本节只需建立印象:

  • ✅ 以后想统一调整颜色与风格,优先来这里修改这些变量
  • ✅ 不需要在整份CSS中逐行寻找所有颜色值

(三)模块样式的大致分区

styles.css
/* header 区域样式 */
header { ... }
.navbar { ... }
/* hero 区域样式 */
#hero { ... }
.hero-title { ... }
/* about 区域样式 */
#about { ... }
.about-content { ... }
/* projects 区域样式 */
#projects { ... }
.project-card { ... }
/* contact 区域样式 */
#contact { ... }
.contact-form { ... }

💡 说明要点:

  • ✅ 每一段注释下面,是对应模块的布局与细节样式
  • ✅ 以后若需要微调某个模块的间距或对齐,可以在相应分区内调整
  • ✅ 本节不要求修改样式,只需要知道样式的大致组织方式

(四)本节的边界提示

需要记住的信息只有两点:
1
想改页面内容
→
回到 index.html 中对应的 <section>
2
想改整体风格
→
在Trae中打开CSS文件,优先查看 :root 中的变量配置

🏷️ 五、教学版模板中的"固定/可改"标注说明

教学版模板中,会通过注释和提示文字标明哪些是固定结构,哪些是可修改内容

HTML中的标注示例 
<!-- 固定结构:建议保留,删除可能导致页面错乱 -->
<section id="projects" class="section section--projects">
  <div class="section__inner">
    <!-- 可修改内容:标题文字可微调 -->
    <h2>项目展示</h2>

    <!-- 固定结构:项目卡片容器 -->
    <div class="projects-grid">
      <!-- 可修改内容:整个项目卡片可增减或替换 -->
      <article class="project-card">
        <!-- 可修改内容:项目标题、描述、标签等 -->
        <h3>示例项目名称</h3>
        <p>示例项目描述...</p>
        <ul class="project-tags">
          <li>HTML</li>
          <li>CSS</li>
        </ul>
      </article>
    </div>
  </div>
</section>
CSS中的标注示例 
/* ===== 固定结构:布局与定位 ===== */
.section {
  padding: 4rem 0;
  max-width: 1200px;
  margin: 0 auto;
}

/* ===== 可修改区域:颜色与视觉效果 ===== */
:root {
  --primary-color: #3498db;      /* 可改:主色 */
  --secondary-color: #2ecc71;    /* 可改:辅色 */
  --border-radius: 8px;         /* 可改:圆角大小 */
}

💡 提示:这些标注在实际使用时会被删除,仅用于教学演示

🚀 六、下节课实战预告

(一)内容替换方向

文本内容替换

  • About模块:替换为个人简介(100–150字)
  • Skills模块:替换为真实技能列表(3–6条)
  • Projects模块:替换为真实项目信息(至少3个)

图片资源替换

  • 头像替换:assets/img/avatar.png
  • 项目封面:assets/img/project-1.png等

链接与联系信息

  • 邮箱、社交链接等
  • 项目详情页或源码链接

(二)风格调整方向

1

颜色主题

通过修改CSS变量快速更换主色、辅色。保持整体协调性,避免过多颜色

2

字体与排版

调整字体大小、行高、字重。保持良好的可读性

3

卡片与组件样式

调整圆角、阴影、边框。保持视觉一致性

(三)常见问题与排查

图片不显示

  • 检查路径与文件名
  • 确认文件已上传至GitHub

样式不生效

  • 检查CSS语法错误
  • 确认选择器与类名匹配

布局错乱

  • 检查HTML标签是否正确闭合
  • 确认未意外删除关键结构
